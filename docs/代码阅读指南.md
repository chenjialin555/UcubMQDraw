## 一、一句话核心

```text
登录拿到 token → 后端从 token 知道你是谁 → 任务写 user_id → WS 只推你的任务 → 前端 upsertTask
```

用户模块就这一句。JWT、游客、拦截器都是为它服务。

WebSocket 就这一句：

```text
连上 WS → 收到 message.task → taskStore.upsertTask
```

重连、心跳、游客过滤都是后面加的。

## 四、正式版分三期（实现与阅读都对齐）

### Phase 1：可读正式版（先实现、先阅读）

**目标：** 注册登录 + 任务隔离 + WS 定向推送，代码保持短、文件拆小。

| 模块 | 做 | 不做 |
|------|-----|------|
| 用户 | register / login / guest / me | 改昵称、改密码、logout 接口 |
| 任务 | POST 创建、GET 列表、GET 单条 | DELETE、rerun、cancel、favorite |
| WS | task.* 事件 + upsertTask | favorite_set、心跳 |
| 前端 WS | 最小连接 + 断线重连 + 重连后 fetchTasks | 游客过滤、心跳 |
| 后端 WS | `send_to_user` 内存版 | Redis Pub/Sub |

前端 WS 目录（Phase 1 可先合并，Phase 2 再拆）：

```text
frontend/src/ws/
├── taskWsMessages.ts    # 消息类型 + parse
├── taskWsHandlers.ts    # 收到消息 → 更新 store
└── taskWsClient.ts      # 连接 + 重连
```

### Phase 2：体验增强

```text
修改昵称 / 改密码
DELETE / rerun / cancel / favorite
游客 localStorage guest_task_ids 过滤
taskStore 筛选 Tab、分页
WS 拆文件 + 心跳
```

### Phase 3：生产增强

```text
Redis Pub/Sub 多实例 WS
登录限流
refreshToken
ws ticket / Cookie 鉴权
审计日志
```

分期详情 → [vue重构方案.md](./vue重构方案.md#三正式版分三期)、[用户与鉴权设计.md](./details/用户与鉴权设计.md#二分期实现)。

## 五、前端文件职责（避免一个文件干十件事）

| 文件 | 只做一件事 |
|------|------------|
| `api/http.ts` | axios + 自动带 Authorization |
| `api/auth.ts` | 注册 / 登录 / guest / me |
| `api/task.ts` | 任务 HTTP |
| `stores/authStore.ts` | token + user 状态 |
| `stores/taskStore.ts` | 任务列表 upsert / fetch |
| `ws/taskWsMessages.ts` | 解析 JSON、类型 |
| `ws/taskWsHandlers.ts` | message → store 动作 |
| `ws/taskWsClient.ts` | 连接、重连、onopen 调 fetchTasks |
| `utils/guestTasks.ts` | Phase 2：游客 taskId 本地集合 |
| `bootstrap.ts` | 启动：guest/login → connect WS |

## 六、我只关心「创建任务后右栏怎么更新」

读这一条路径即可：

```text
1. HomePage submit → taskStore.createTask
2. POST /tasks（http 自动带 token）
3. 后端写 DB，send_to_user 推 task.created
4. ws onmessage → upsertTask
5. TaskList 自动重绘
```

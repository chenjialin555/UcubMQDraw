## 一、启动

```bash
# 后端
cd backend
uv sync
USE_MOCK=true uv run uvicorn app.main:app --reload --port 8000

# 前端
cd frontend
npm install
npm run dev
```

浏览器打开 `http://localhost:5173`，API 指向 `http://localhost:8000`。

环境变量完整列表 → [details/部署与环境变量.md](./details/部署与环境变量.md)

## 三、任务主链路（对着查 bug）

```text
上传图片 → url 写入表单
  ↓
tool.buildRequest() → POST /tasks（见 API接口.md）
  ↓
validate → 写 DB → Mock 线程 / MQ
  ↓
imggen 执行 → 回调 → WebSocket → 右栏更新
```

状态细节 → [details/任务状态与任务流转.md](./details/任务状态与任务流转.md)

## 四、Mock 模式（开发必用）

```env
USE_MOCK=true
```

不投 MQ，后台线程模拟进度，右栏能看到完整进度条。

详见 [details/Mock模式设计.md](./details/Mock模式设计.md)

## 五、WebSocket 右栏刷新

```text
main.ts 启动 connectTaskWebSocket()
  → 收到 task_update
  → taskStore.upsertTask(task)
  → TaskList 自动重绘
```

右栏挂载时务必 `fetchTasks()` 拉历史。详见 [details/WebSocket推送机制.md](./details/WebSocket推送机制.md)

## 六、常见问题

### 提交成功但右栏没任务

- 检查 `taskStore.createTask` 是否把返回任务插入列表
- 检查 WS 是否连接（浏览器 Network → WS）

### 图片上传后预览裂图

- 确认 OSS AccessKey、Bucket、Endpoint 配置正确
- 确认返回 URL 可公网访问（`public-read` + 自定义域名）
- 确认 imggen 能访问同一 OSS 域名

### Mock 进度不走

- 确认 `USE_MOCK=true` 且后端日志有 `start_mock_task`

### POST /tasks 400

- 对照 [API接口.md](./API接口.md) 看 params 结构
- 对照后端 `XxxTool.validate` 错误信息

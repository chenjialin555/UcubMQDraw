## Mock / Stub 统一原则

UcubMQDraw **不在业务侧伪造 imggen 进度**。本地没有真实 imggen 时，启动独立的 [imggen-stub](./details/imggen-stub联调模式设计.md)，仍走完整 MQ 链路。

## 一、第一次启动

### 前置

- Docker Desktop（PostgreSQL + Redis）
- 可连通的阿里云 RocketMQ（本地联调必填）
- Node.js 18+（前端）
- Python 3.11+ / uv（后端 + imggen-stub）

### 启动步骤

```bash
# 1. 启动本地基础服务（PostgreSQL + Redis）
docker compose up -d
docker compose ps

# 2. 配置后端环境变量
cp backend/.env.example backend/.env
# 填写 OSS；填写 RocketMQ（本地联调也要配，不再有 USE_MOCK）

# 3. 后端
cd backend
uv sync
uv run uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

# 4. imggen-stub（假生图，走真实 MQ）
cd imggen-stub
cp .env.example .env   # Topic 与后端一致
uv run python -m app.main

# 5. 前端
cd frontend
npm install
npm run dev
```

浏览器打开 `http://localhost:5173`，API 指向 `http://localhost:8000`。

### 服务分工

```text
本机运行：Vue 前端、FastAPI 后端、imggen-stub
Docker：  PostgreSQL、Redis
阿里云：  OSS、RocketMQ（外部服务，不在 docker-compose 里）
```

环境变量完整列表 → [details/部署与环境变量.md](./details/部署与环境变量.md)

### 检查 Docker 是否正常

```bash
docker compose ps
# postgres / redis 状态应为 healthy

docker compose logs postgres --tail 20
docker compose logs redis --tail 20
```

PostgreSQL 首次启动会自动执行 `docker/postgres/init/01_init.sql` 建表。

## 三、任务主链路（对着查 bug）

```text
上传图片 → url 写入表单
  ↓
tool.buildRequest() → POST /tasks（见 API接口.md）
  ↓
validate → 写 PostgreSQL → 投递 RocketMQ
  ↓
imggen 或 imggen-stub 消费 → 回调 MQ
  ↓
Callback Consumer 更新 DB → WebSocket → 右栏更新
```

观察点：

```text
写库 → MQ task → stub/imggen callback → DB 更新 → WS
```

状态细节 → [details/任务状态与任务流转.md](./details/任务状态与任务流转.md)

## 五、WebSocket 右栏刷新

```text
bootstrap：无 token / token 失效 → POST /auth/guest
main.ts 启动 connectTaskWebSocket(?token=...)
  → 收到 task.* → taskStore.upsertTask / removeTask
  → 重连成功后 fetchTasks（非游客）
```

鉴权详见 [details/用户与鉴权设计.md](./details/用户与鉴权设计.md)。WS 最小示例 → [WebSocket推送示例.md](./examples/WebSocket推送示例.md#二最小可读版先看这个)。

右栏挂载时务必 `fetchTasks()` 拉历史。详见 [details/WebSocket推送机制.md](./details/WebSocket推送机制.md)

## 六、常见问题

### docker compose 启动失败

- 确认 Docker Desktop 已运行
- 5432 / 6379 端口未被占用
- 查看 `docker compose logs postgres`

### 后端连不上数据库

- 确认 `docker compose ps` 中 postgres 为 healthy
- 确认 `DATABASE_URL=postgresql+psycopg://ucub:ucub_password@localhost:5432/ucub_mqdraw`

### 提交成功但右栏没任务

- 检查 `taskStore.createTask` 是否把返回任务插入列表
- 检查 WS 是否连接（浏览器 Network → WS）

### 图片上传后预览裂图

- 确认 OSS AccessKey、Bucket、Endpoint 配置正确
- 确认返回 URL 可公网访问

### 任务一直 queued / 没有 succeeded

- 确认 RocketMQ 配置完整、`mq_enabled()` 为真
- 确认 imggen-stub（或真实 imggen）已启动并订阅同一 `ROCKETMQ_TOPIC_TASK`
- 确认 UcubMQDraw Callback Consumer 已启动并监听 `ROCKETMQ_TOPIC_CALLBACK`

### POST /tasks 400

- 对照 [API接口.md](./API接口.md) 看 params 结构
- 对照后端 `XxxTool.validate` 错误信息

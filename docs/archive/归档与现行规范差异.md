## 一、怎么用本文

```text
以现行 docs 为准
归档仅作历史背景或第二版参考
```

## 二、关键差异对照

| 主题 | 归档写法 | 现行规范 | 处理 |
|------|----------|----------|------|
| 本地存储 | `USE_LOCAL_STORAGE` 开关 | 统一 OSS URL | **已废弃**，见 [OSS上传设计.md](../details/OSS上传设计.md) |
| 前端目录 | `features/` | `tools/` | **已改名**，见 [features-to-tools术语对照.md](./features-to-tools术语对照.md) |
| 任务状态终态 | `completed` | `succeeded` | **以 API 为准** |
| WebSocket | 全局广播 | 按 `user_id` 定向推送 | **正式版必须定向** |
| 任务队列 | Redis List 等备选 | RocketMQ（阿里云） | **队列只用 RocketMQ** |
| 数据库 | SQLite 开发 / MySQL 生产 | PostgreSQL 统一 | 见 [数据库存储设计.md](../details/数据库存储设计.md) |
| Redis | 常与任务队列混写 | 仅 WS Pub/Sub、限流（本地 Docker） | **不是任务队列** |
| 任务操作 | retry / cancel / favorite 混在 MVP | Phase 2 再做 | MVP 只做创建 + 查询 |
| 用户鉴权 | 无或简略 | JWT + `user_id` 隔离 | Phase 1 必做，见 [用户与鉴权设计.md](../details/用户与鉴权设计.md) |
| HTTP 回调 imggen | 部分章节仍写 HTTP callback | MQ 回调为主 | 以 [RocketMQ对接设计.md](../details/RocketMQ对接设计.md) 为准 |
| taskStore 完整版 | 归档含分页/收藏/筛选大全 | 示例分期，先最小链路 | 见 [WebSocket推送示例.md](../examples/WebSocket推送示例.md) §二 |

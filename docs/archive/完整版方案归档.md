# UcubMQDraw 前后端重构完整方案

## 一、重构目标

当前项目是一个 Gradio 前端壳，主要职责包括：

- 左侧表单采集
- 中间图片预览和蒙版编辑
- 任务提交
- 本地内存 Store
- WebSocket 状态监听
- 右侧任务记录展示

重构后目标是：

```text
Vue3 前端
  负责页面交互、表单、预览、任务列表展示、WebSocket 实时更新

FastAPI 后端
  负责任务创建、图片上传、任务持久化、MQ 投递、状态更新、WebSocket 推送

RocketMQ / imggen / ComfyUI
  仍由后端对接，前端完全不感知
```

最终架构：

```text
用户
 ↓
Vue3 前端
 ↓ HTTP / WebSocket
FastAPI 后端
 ↓
数据库 / OSS / RocketMQ
 ↓
imggen / ComfyUI
```

# 四、核心架构原则

## 4.1 前端原则

前端只做这几件事：

```text
1. 渲染页面
2. 管理用户输入
3. 上传图片
4. 创建任务
5. 展示任务列表
6. 接收 WebSocket 更新
```

前端不做：

```text
1. 不投递 MQ
2. 不生成 innerTaskId
3. 不维护长期任务状态
4. 不做 OSS placeholder
5. 不直接接触 imggen / ComfyUI
6. 不传 computeType（算力/MQ 路由由后端根据 toolKey 内部决定）
```

---

## 4.2 后端原则

后端负责完整任务生命周期：

```text
1. 接收创建任务请求
2. 校验不同功能的参数
3. 生成 taskId / innerTaskId
4. 保存任务到数据库
5. 投递 RocketMQ
6. 接收 imggen 回调
7. 更新任务状态
8. 通过 WebSocket 推送给前端
```

---

## 4.3 功能模块原则

每个功能独立成一个模块。

例如：

```text
风格迁移:
  表单字段不同
  校验逻辑不同
  payload 不同
  workflow 不同

局部重绘:
  需要蒙版
  需要重绘强度
  需要蒙版羽化
  payload 里有 maskImageUrl
```

所以不能把所有逻辑都塞在 `HomePage.vue` 或 `task_service.py` 里。

采用：

```text
前端 Feature Definition
后端 Feature Handler
```

### 5.2.1 统一 Feature 类型

`src/features/types.ts`

```ts
import type { Component } from 'vue'
import type { CreateTaskRequest } from '@/api/task'

/** 表单校验结果，valid=false 时 message 用于 ElMessage 提示 */
export interface ValidateResult {
  valid: boolean
  message?: string
}

/**
 * 功能模块统一接口
 * 每个创作工具（风格迁移、局部重绘等）实现一份 definition
 */
export interface FeatureDefinition<TForm> {
  key: string                          // 稳定标识，对应后端 toolKey
  name: string                         // 中文展示名
  description: string                  // 工具说明文案

  formComponent: Component             // 左栏表单组件
  previewComponent?: Component         // 中栏预览组件（可选）

  createDefaultForm: () => TForm       // 创建表单初始值
  validateForm: (form: TForm) => ValidateResult   // 提交前校验
  buildPayload: (form: TForm) => CreateTaskRequest // 构建 API 请求体
}
```

这里故意保持简单。

一个功能只需要提供：

```text
1. key
2. name
3. 表单组件
4. 可选预览组件
5. 默认表单
6. 校验方法
7. payload 构建方法
```

### 5.3.4 校验和 payload

`src/features/style-transfer/submit.ts`

```ts
import type { CreateTaskRequest } from '@/api/task'
import type { ValidateResult } from '../types'
import type { StyleTransferFormData } from './types'

/** 返回风格迁移表单的初始默认值 */
export function createDefaultForm(): StyleTransferFormData {
  return {
    baseImageUrl: '',
    styleImageUrl: '',
    generationCount: 1,
    styleStrength: 0.75,
    description: '',
  }
}

/**
 * 校验风格迁移表单必填项
 * @returns valid=false 时 message 为前端提示文案
 */
export function validateForm(form: StyleTransferFormData): ValidateResult {
  if (!form.baseImageUrl) {
    return { valid: false, message: '请上传底图' }
  }

  if (!form.styleImageUrl) {
    return { valid: false, message: '请上传风格参考图' }
  }

  if (form.description.length > 300) {
    return { valid: false, message: '描述不能超过 300 个字符' }
  }

  return { valid: true }
}

/**
 * 将表单数据转换为创建任务 API 请求体
 * 各功能参数放入 params，结构由功能自己定义（文生图可无 refImageList）
 */
export function buildPayload(form: StyleTransferFormData): CreateTaskRequest {
  return {
    toolKey: 'style_transfer',
    params: {
      prompt: form.description || '风格迁移',
      refImageList: [
        form.baseImageUrl,
        form.styleImageUrl,
      ],
      generationCount: form.generationCount,
      styleStrength: form.styleStrength,
    },
  }
}
```

```vue
  if (!feature) {
    return

  currentForm.value = feature.createDefaultForm()

 * 提交当前表单
```

# 七、前端状态管理

`src/stores/taskStore.ts`

```ts
import { defineStore } from 'pinia'
import type { GenerationTask } from '@/types/task'
import {
  listTasks,
  createTask,
  retryTask,
  cancelTask,
  favoriteTask,
  type CreateTaskRequest,
} from '@/api/task'

interface TaskState {
  tasks: GenerationTask[]
  currentFilter: string
  loading: boolean
}

const runningStatuses = [
  'created',
  'queued',
  'dispatching',
  'running',
  'retrying',
]

export const useTaskStore = defineStore('task', {
  state: (): TaskState => ({
    tasks: [],
    currentFilter: '全部',
    loading: false,
  }),

  getters: {
    /** 根据 currentFilter 筛选任务列表 */
    filteredTasks(state) {
      if (state.currentFilter === '全部') {
        return state.tasks
      }

      if (state.currentFilter === '进行中') {
        return state.tasks.filter(task =>
          runningStatuses.includes(task.status),
        )
      }

      if (state.currentFilter === '已完成') {
        return state.tasks.filter(task => task.status === 'completed')
      }

      if (state.currentFilter === '收藏') {
        return state.tasks.filter(task => task.favorite)
      }

      return state.tasks
    },
  },

  actions: {
    /** 从后端拉取全部任务列表 */
    async fetchTasks() {
      this.loading = true

      try {
        this.tasks = await listTasks()
      } finally {
        this.loading = false
      }
    },

    /** 设置右栏筛选 Tab */
    setFilter(filter: string) {
      this.currentFilter = filter
    },

    /**
     * 插入或更新单条任务（WebSocket 推送和 HTTP 提交共用）
     * 已存在则合并字段，不存在则插入列表头部
     */
    upsertTask(task: GenerationTask) {
      const index = this.tasks.findIndex(item => item.taskId === task.taskId)

      if (index >= 0) {
        this.tasks[index] = {
          ...this.tasks[index],
          ...task,
        }
      } else {
        this.tasks.unshift(task)
      }
    },

    /** 创建任务并写入本地列表 */
    async createTask(data: CreateTaskRequest) {
      const task = await createTask(data)
      this.upsertTask(task)
      return task
    },

    /** 重跑任务，后端会创建新 taskId */
    async retry(taskId: string) {
      const task = await retryTask(taskId)
      this.upsertTask(task)
    },

    /** 取消任务 */
    async cancel(taskId: string) {
      const task = await cancelTask(taskId)
      this.upsertTask(task)
    },

    /** 切换收藏状态 */
    async setFavorite(taskId: string, favorite: boolean) {
      const task = await favoriteTask(taskId, favorite)
      this.upsertTask(task)
    },
  },
})
```

```ts
import { useTaskStore } from '@/stores/taskStore'

let socket: WebSocket | null = null
let reconnectTimer: number | null = null

/**
 * 建立任务状态 WebSocket 连接
 * 收到 task_update 消息时自动 upsert 到 Pinia Store
 * 断线后 3 秒自动重连
 */
export function connectTaskWebSocket() {
  const taskStore = useTaskStore()
  const wsUrl = import.meta.env.VITE_WS_URL

  if (socket) {
    socket.close()
  }

  socket = new WebSocket(wsUrl)

  socket.onmessage = event => {
    try {
      const data = JSON.parse(event.data)

      if (data.type === 'task_update' && data.task) {
        taskStore.upsertTask(data.task)
      }
    } catch (error) {
      console.error('[WS] invalid message', error)
    }
  }

  socket.onclose = () => {
    if (reconnectTimer) {
      window.clearTimeout(reconnectTimer)
    }

    reconnectTimer = window.setTimeout(() => {
      connectTaskWebSocket()
    }, 3000)
  }

  socket.onerror = error => {
    console.error('[WS] error', error)
  }
}
```

# 十、后端数据模型

## 10.0 参数字段设计原则

每个生图功能的请求参数不同（风格迁移要两张图、局部重绘要蒙版、文生图可能只要 prompt），**不适合为每种参数建一列扁平字段**。

采用「**固定列 + 三个 JSON 列**」（命名规则：`task_{阶段}_params`）：

| 阶段 | DB 列名 | API camelCase | 读法 | 存什么 | 谁写 | 谁读 |
|------|---------|---------------|------|--------|------|------|
| 用户提交 | `task_submit_params` | `taskSubmitParams` | 任务·提交·参数 | 前端 `params` 原文 | 创建任务 | 重跑、右栏输入预览 |
| 投递算力 | `task_dispatch_params` | —（仅落库，不返前端） | 任务·下发·参数 | Handler 组装后投 MQ 的 JSON | Handler | 排错、对账 |
| 算力回调 | `task_callback_params` | `taskCallbackParams` | 任务·回调·参数 | imggen 回调完整 JSON | 回调 Consumer | 右栏输出预览 |

**不再单独存：**

```text
workflow_template   → 若 imggen 需要，写在 task_dispatch_params 内部，Handler 私有常量
prompt / ref_image_list / extra / mask_image_url / mq_payload ...  → 全部并入上述 JSON
input_preview / output_preview  → 右栏 UI 从 task_submit_params / task_callback_params 按需解析
```

文生图等无底图场景：`task_submit_params` 里无 `refImageList`，`TaskCard` 显示占位图即可。

# 十三、后端 Service 核心逻辑

`backend/app/services/task_service.py`

```python
from app.features.registry import get_feature_handler
from app.repositories.task_repository import task_repository
from app.services.mq_service import mq_service
from app.services.websocket_manager import websocket_manager
from app.services.mock_task_runner import start_mock_task
from app.utils.id_generator import generate_task_id, generate_inner_task_id
from app.config import settings


class TaskService:
    """任务生命周期服务：创建、查询、重跑、取消、收藏"""

    def create_task(self, data: dict):
        """
        创建生图任务完整流程：
        1. 根据 toolKey 获取 Handler 并校验
        2. 生成 taskId / innerTaskId
        3. 持久化到数据库
        4. 投递 MQ 或启动 Mock
        5. WebSocket 广播初始状态
        """
        tool_key = data["toolKey"]
        handler = get_feature_handler(tool_key)
        task_submit_params = data.get("params") or {}

        handler.validate(data)

        task_id = generate_task_id()
        inner_task_id = generate_inner_task_id()

        task_dispatch_params = handler.build_task_dispatch_params(
            task_id=task_id,
            inner_task_id=inner_task_id,
            params=task_submit_params,
        )

        task = task_repository.create(
            task_id=task_id,
            inner_task_id=inner_task_id,
            tool_key=handler.key,
            tool_name=handler.name,
            status="created",
            progress=0,
            task_submit_params=task_submit_params,
            task_dispatch_params=task_dispatch_params,
        )

        if settings.USE_MOCK:
            task = task_repository.update_status(
                task_id=task_id,
                status="queued",
                progress=5,
            )
            start_mock_task(task_id)
        else:
            mq_service.send(task_dispatch_params, tool_key=handler.key)
            task = task_repository.update_status(
                task_id=task_id,
                status="queued",
                progress=5,
            )

        websocket_manager.broadcast_task_update(task)

        return task

    def list_tasks(self):
        """返回全部任务列表，按创建时间倒序"""
        return task_repository.list()

    def get_task(self, task_id: str):
        """根据业务 taskId 查询单条任务"""
        return task_repository.get_by_task_id(task_id)

    def retry_task(self, task_id: str):
        """
        基于原任务 task_submit_params 创建新任务
        第一版策略：生成新 taskId，不覆盖旧任务
        """
        old_task = task_repository.get_by_task_id(task_id)

        if not old_task:
            raise ValueError("任务不存在")

        return self.create_task({
            "toolKey": old_task.tool_key,
            "params": old_task.task_submit_params,
        })

    def cancel_task(self, task_id: str):
        """将任务标记为 cancelled 并推送 WebSocket"""
        task = task_repository.update_status(
            task_id=task_id,
            status="cancelled",
            progress=0,
        )

        websocket_manager.broadcast_task_update(task)

        return task

    def favorite_task(self, task_id: str, favorite: bool):
        """切换收藏状态并推送 WebSocket"""
        task = task_repository.update_favorite(task_id, favorite)
        websocket_manager.broadcast_task_update(task)
        return task


task_service = TaskService()
```

## 15.4 OSS Client

`backend/app/clients/oss_client.py`

```python
import logging
import os
import uuid
from datetime import datetime
from pathlib import Path

import alibabacloud_oss_v2 as oss

from app.config import settings, oss_enabled

logger = logging.getLogger(__name__)


class OSSUploadError(Exception):
    """OSS 上传失败异常，便于 API 层统一捕获"""

    def __init__(self, message: str, object_name: str | None = None):
        super().__init__(message)
        self.message = message
        self.object_name = object_name


class OSSClient:
    """
    阿里云 OSS 上传客户端（简化版）。
    仅封装单文件上传，供 upload_service 调用。
    """

    def __init__(self):
        self._client = None
        if oss_enabled():
            self._client = self._build_client()

    def _build_client(self) -> oss.Client:
        """初始化 OSS SDK 客户端"""
        credentials = oss.credentials.StaticCredentialsProvider(
            settings.OSS_ACCESS_KEY_ID,
            settings.OSS_ACCESS_KEY_SECRET,
        )
        cfg = oss.config.load_default()
        cfg.credentials_provider = credentials
        cfg.region = settings.OSS_REGION
        cfg.endpoint = settings.OSS_ENDPOINT
        cfg.use_cname = settings.OSS_USE_CNAME
        return oss.Client(cfg)

    def build_object_key(self, filename: str) -> str:
        """
        生成 OSS 对象键
        格式：{OSS_PATH}/{yyyyMMdd}/{uuid8}_{filename}
        例：ai/img/build/20260712/a1b2c3d4_base.png
        """
        date_part = datetime.now().strftime("%Y%m%d")
        safe_name = os.path.basename(filename) or "image.png"
        unique = uuid.uuid4().hex[:8]
        return f"{settings.OSS_PATH}/{date_part}/{unique}_{safe_name}"

    def build_public_url(self, object_key: str) -> str:
        """
        拼接公网访问 URL
        use_cname=true 时：{endpoint}/{object_key}
        """
        endpoint = settings.OSS_ENDPOINT.rstrip("/")
        return f"{endpoint}/{object_key}"

    def upload(
        self,
        filename: str,
        content: bytes,
        content_type: str = "image/png",
    ) -> str:
        """
        上传文件并返回可访问 URL。
        - OSS 已配置：put_object + acl=public-read
        - 未配置：写入本地 static/uploads
        """
        if not oss_enabled() or settings.USE_LOCAL_STORAGE:
            return self._upload_local(filename, content)

        object_key = self.build_object_key(filename)

        try:
            result = self._client.put_object(
                oss.PutObjectRequest(
                    bucket=settings.OSS_BUCKET_NAME,
                    key=object_key,
                    body=content,
                    headers={"Content-Type": content_type},
                    acl="public-read",          # 桶策略允许时，对象可被前端/imggen 直接读取
                )
            )
            if not (result and getattr(result, "status_code", None) == 200):
                raise OSSUploadError(
                    f"OSS 上传失败: status={getattr(result, 'status_code', None)}",
                    object_name=object_key,
                )

            url = self.build_public_url(object_key)
            logger.info("OSS 上传成功: object=%s url=%s", object_key, url)
            return url

        except OSSUploadError:
            raise
        except Exception as exc:
            raise OSSUploadError(
                f"OSS 上传异常: {exc}",
                object_name=object_key,
            ) from exc

    def _upload_local(self, filename: str, content: bytes) -> str:
        """开发回退：保存到本地目录，返回 /static/uploads/... 路径"""
        upload_dir = Path(settings.LOCAL_UPLOAD_DIR)
        upload_dir.mkdir(parents=True, exist_ok=True)

        object_key = self.build_object_key(filename)
        local_path = upload_dir / object_key.replace("/", "_")
        local_path.parent.mkdir(parents=True, exist_ok=True)
        local_path.write_bytes(content)

        url = f"/static/uploads/{local_path.name}"
        logger.info("本地存储上传: path=%s url=%s", local_path, url)
        return url


oss_client = OSSClient()
```

要点说明：

```text
1. 使用 alibabacloud_oss_v2 + StaticCredentialsProvider，与现有项目一致
2. use_cname=true + 自定义 endpoint，URL 直接拼 {endpoint}/{object_key}
3. acl=public-read，确保 imggen / 前端预览能匿名读取（桶 ACL 需配合）
4. 对象路径统一走 OSS_PATH 前缀，便于按业务隔离
5. 开发环境 OSS 未配或 USE_LOCAL_STORAGE=true 时，自动回退本地
6. 不做 batch_upload / AIGC 元数据注入 / 桶自动创建，第一版保持简单
```

```python
import json
from fastapi import WebSocket


class WebSocketManager:
    """管理所有前端 WebSocket 连接，广播任务状态变更"""

    def __init__(self):
        self.connections: list[WebSocket] = []

    async def connect(self, websocket: WebSocket):
        """接受新连接并加入连接池"""
        await websocket.accept()
        self.connections.append(websocket)

    def disconnect(self, websocket: WebSocket):
        """从连接池移除断开的连接"""
        if websocket in self.connections:
            self.connections.remove(websocket)

    async def broadcast(self, data: dict):
        """向所有在线客户端推送 JSON 消息，自动清理失效连接"""
        disconnected = []

        for websocket in self.connections:
            try:
                await websocket.send_text(json.dumps(data, ensure_ascii=False))
            except Exception:
                disconnected.append(websocket)

        for websocket in disconnected:
            self.disconnect(websocket)

    def broadcast_task_update(self, task):
        """
        封装任务更新消息并广播
        消息格式：{"type": "task_update", "task": {...}}
        注意：同步上下文中调用需通过 event loop 调度
        """
        data = {
            "type": "task_update",
            "task": task.to_response_dict(),
        }

        # 实际代码里这里需要放到 event loop 中执行
        # 可以用 background task 或 anyio.from_thread.run
```

实际项目中要注意：如果在普通同步函数里广播，需要处理 async event loop。第一版可以把 task service 做成 async，或者通过后台任务统一推送。

## 17.3 RocketMQ Producer Client

`backend/app/clients/rocketmq_client.py`

参考现有项目的 `RocketMQProducer`，简化为单 Topic 发送 + 指数退避重试：

```python
import json
import logging
import time
import uuid

from rocketmq.client import Message, Producer

from app.config import settings, mq_enabled

logger = logging.getLogger(__name__)


class RocketMQSendError(Exception):
    """MQ 发送失败异常"""

    def __init__(self, message: str, topic: str | None = None):
        super().__init__(message)
        self.message = message
        self.topic = topic


class RocketMQClient:
    """
    RocketMQ 生产者客户端（单例）。
    封装同步发送、ACL 鉴权、失败重试。
    """

    _instance = None

    def __new__(cls, *args, **kwargs):
        if not cls._instance:
            cls._instance = super().__new__(cls, *args, **kwargs)
            cls._instance._init_producer()
        return cls._instance

    def _init_producer(self) -> None:
        """初始化 Producer，未配置 MQ 时跳过"""
        self._producer = None
        if not mq_enabled():
            logger.warning("RocketMQ 未配置，Producer 不启动")
            return

        # 组名加随机后缀，避免多实例冲突
        group_name = f"{settings.ROCKETMQ_PRODUCER_GROUP}-{uuid.uuid4().hex[:8]}"

        producer = Producer(group_name)
        producer.set_name_server_address(settings.ROCKETMQ_NAME_SERVER)
        producer.set_session_credentials(
            settings.ROCKETMQ_ACCESS_KEY,
            settings.ROCKETMQ_SECRET_KEY,
            "",
        )
        producer.start()
        self._producer = producer
        logger.info("RocketMQ Producer 启动: group=%s", group_name)

    def send_message(
        self,
        topic: str,
        body: str,
        *,
        tags: str = "",
        keys: str = "",
    ):
        """
        同步发送消息，失败时指数退避重试。

        :param topic: 目标 Topic
        :param body: 消息体（JSON 字符串）
        :param tags: 消息标签，建议传 toolKey
        :param keys: 消息键，建议传 innerTaskId / taskId
        :returns: 发送结果对象
        """
        if not self._producer:
            raise RocketMQSendError("RocketMQ 未初始化", topic=topic)

        msg = Message(topic)
        msg.set_keys(keys)
        msg.set_tags(tags)
        msg.set_body(body)

        max_retries = settings.ROCKETMQ_RETRY_TIMES
        backoff_base = settings.ROCKETMQ_RETRY_BACKOFF_BASE
        backoff_max = settings.ROCKETMQ_RETRY_BACKOFF_MAX

        last_error = None
        for attempt in range(1, max_retries + 2):  # 首次 + 重试次数
            try:
                result = self._producer.send_sync(msg)
                if getattr(result, "status", -1) == 0:
                    logger.info(
                        "MQ 发送成功: topic=%s keys=%s msg_id=%s attempt=%d",
                        topic, keys, getattr(result, "msg_id", None), attempt,
                    )
                    return result

                last_error = RuntimeError(f"status={getattr(result, 'status', None)}")
                logger.warning("MQ 返回非成功状态，准备重试: attempt=%d", attempt)

            except Exception as exc:
                last_error = exc
                logger.warning("MQ 发送异常，准备重试: attempt=%d err=%s", attempt, exc)

            if attempt <= max_retries:
                sleep_sec = min(backoff_base * (2 ** (attempt - 1)), backoff_max)
                time.sleep(sleep_sec)

        raise RocketMQSendError(
            f"MQ 发送失败: {last_error}",
            topic=topic,
        ) from last_error

    def shutdown(self) -> None:
        """应用关闭时释放 Producer"""
        if self._producer:
            self._producer.shutdown()
            logger.info("RocketMQ Producer 已关闭")


rocketmq_client = RocketMQClient()
```

要点说明：

```text
1. 单例 Producer，启动时 set_name_server_address + set_session_credentials
2. send_sync 同步发送，status==0 为成功
3. 失败指数退避重试，次数由 ROCKETMQ_RETRY_TIMES 控制
4. 不做 Consumer / 批量发送，第一版保持简单
5. USE_MOCK=true 时 task_service 不调用本客户端
```

```python
import json
import logging

from app.clients.rocketmq_client import rocketmq_client, RocketMQSendError
from app.config import settings, mq_enabled

logger = logging.getLogger(__name__)


class MQService:
    """
    MQ 业务封装层。
    负责：选 Topic、序列化 payload、调用 Producer。
    """

    def send(self, payload: dict, tool_key: str) -> None:
        """
        投递任务消息到 imggen 任务 Topic。

        :param task_dispatch_params: Handler 构建的完整 imggen 请求 JSON
        :param tool_key: 功能标识，用作 tags 和 Topic 路由

        if not mq_enabled():
            raise RocketMQSendError("RocketMQ 未配置，无法投递")

        topic = self._resolve_task_topic(tool_key)
        body = json.dumps(payload, ensure_ascii=False)
        keys = str(payload.get("taskId") or payload.get("bizTaskId") or "")

        rocketmq_client.send_message(
            topic=topic,
            body=body,
            tags=tool_key,
            keys=keys,
        )

    def _resolve_task_topic(self, tool_key: str) -> str:
        """
        根据 toolKey 选择下发 Topic。
        第一版所有工具走同一 Topic；后续可按 tool_key 扩展多 Topic。
        """
        _ = tool_key  # 预留路由扩展点
        return settings.ROCKETMQ_TOPIC_TASK


mq_service = MQService()
```

## 17.5 回调 Consumer（简化版）

`backend/app/clients/rocketmq_consumer.py`

```python
import json
import logging

from rocketmq.client import PushConsumer

from app.config import settings, mq_enabled
from app.services.callback_service import handle_imggen_callback

logger = logging.getLogger(__name__)


def start_callback_consumer() -> None:
    """
    启动 imggen 回调 Consumer（后台线程）。
    在 FastAPI lifespan 启动时调用；USE_MOCK 或未配置 MQ 时跳过。
    """
    if not mq_enabled():
        logger.warning("RocketMQ 未配置，Callback Consumer 不启动")
        return

    consumer = PushConsumer(f"ucub-mqdraw-callback-{settings.ROCKETMQ_TOPIC_CALLBACK}")
    consumer.set_name_server_address(settings.ROCKETMQ_NAME_SERVER)
    consumer.set_session_credentials(
        settings.ROCKETMQ_ACCESS_KEY,
        settings.ROCKETMQ_SECRET_KEY,
        "",
    )
    consumer.subscribe(settings.ROCKETMQ_TOPIC_CALLBACK, _on_message)
    consumer.start()
    logger.info("Callback Consumer 已启动: topic=%s", settings.ROCKETMQ_TOPIC_CALLBACK)


def _on_message(msg):
    """Consumer 回调：解析 JSON → handle_imggen_callback"""
    try:
        body = msg.body.decode("utf-8")
        data = json.loads(body)
        handle_imggen_callback(data)
        return 0  # CONSUME_SUCCESS
    except Exception as exc:
        logger.error("回调消息处理失败: %s", exc, exc_info=True)
        return 1  # RECONSUME_LATER
```

`backend/app/main.py` lifespan 中启动：

```python
from contextlib import asynccontextmanager
from fastapi import FastAPI
from app.clients.rocketmq_consumer import start_callback_consumer
from app.clients.rocketmq_client import rocketmq_client


@asynccontextmanager
async def lifespan(app: FastAPI):
    start_callback_consumer()   # 非 Mock 且 MQ 已配置时启动
    yield
    rocketmq_client.shutdown()


app = FastAPI(lifespan=lifespan)
```

```python
from app.repositories.task_repository import task_repository
from app.services.websocket_manager import websocket_manager
from app.utils.status import normalize_status


def handle_imggen_callback(message: dict) -> None:
    """
    处理 imggen 统一回调消息。
    消息体 taskId 字段为 innerTaskId，需映射到业务 taskId。

    期望格式（见 docs/mq-contract.md）：
    {
      "innerTaskId": "inner_xxx",   # 或 "taskId" 字段
      "status": "success",
      "costTime": 24.3,
      "imageUrlList": ["https://..."],
      "errorMsg": ""
    }
    """
    inner_task_id = message.get("innerTaskId") or message.get("taskId")
    if not inner_task_id:
        return

    task = task_repository.get_by_inner_task_id(inner_task_id)
    if not task:
        return

    updated_task = task_repository.update_result(
        task_id=task.task_id,
        status=normalize_status(message.get("status")),
        progress=100 if message.get("status") in ("success", "completed") else task.progress,
        task_callback_params=message,
        error_message=message.get("errorMsg") or "",
        cost_time=message.get("costTime"),
    )

    websocket_manager.broadcast_task_update(updated_task)
```

# 部署与环境变量

> 基础设施原则：**本地 Docker 只起 PostgreSQL + Redis**；**OSS / RocketMQ 用阿里云外部服务**；FastAPI / Vue 在本机运行。

---

## 一、服务来源

| 服务 | 来源 | 用途 |
|------|------|------|
| PostgreSQL | 本地 Docker Compose | 用户、任务、JSONB 参数 |
| Redis | 本地 Docker Compose | WS 多实例 Pub/Sub、登录限流（Phase 3） |
| OSS | **阿里云外部服务** | 图片上传、生成结果存储 |
| RocketMQ | **阿里云外部服务** | 任务投递、imggen 回调 |
| FastAPI | 本机运行 | API、鉴权、任务、WebSocket |
| Vue | 本机运行 | 前端开发服务 |

```text
Vue / FastAPI  → 本机
PostgreSQL / Redis → Docker Compose
OSS / RocketMQ → 阿里云（不在 docker-compose 中启动）
```

### OSS / RocketMQ 说明

- **OSS**：阿里云 OSS 或兼容 OSS 的对象存储，用于图片上传。
- **RocketMQ**：阿里云 RocketMQ，用于任务投递和 imggen 回调。
- **本地 Docker Compose 不启动 OSS / RocketMQ**。
- 没有真实 imggen 时，启动独立 **imggen-stub** 仍消费同一 Task Topic、投递 Callback Topic；**禁止**用 `USE_MOCK` 跳过 MQ。
  详见 [imggen-stub联调模式设计.md](./imggen-stub联调模式设计.md)。

### Redis 职责（重要）

```text
Redis 不作为任务队列。
任务队列仍然是阿里云 RocketMQ。
```

Redis 用途：

| 用途 | 阶段 | 说明 |
|------|------|------|
| WebSocket 多实例事件分发 | Phase 3 | `WS_EVENT_BUS=redis` |
| 登录限流 | Phase 3 | `AUTH_RATE_LIMIT_BACKEND=redis` |
| 后续缓存 | 待定 | — |

Phase 1 默认 `WS_EVENT_BUS=local`，单实例内存推送，**Redis 容器仍建议启动**（为后续限流/Pub/Sub 预留）。

## 二、本地 Docker 服务

### 目录结构

```text
docker/
├── postgres/
│   ├── data/          # PG 数据卷（git 忽略）
│   └── init/
│       └── 01_init.sql
└── redis/
    └── data/          # Redis 数据卷（git 忽略）
```

### 启动

```bash
# 项目根目录
docker compose up -d
docker compose ps
```

### 查看日志

```bash
docker compose logs -f postgres
docker compose logs -f redis
```

### 停止

```bash
docker compose down
```

### 清空数据（慎用）

```bash
docker compose down -v
# 或手动删除
rm -rf docker/postgres/data/*
rm -rf docker/redis/data/*
```

首次启动 postgres 会自动执行 `docker/postgres/init/01_init.sql` 建表并插入游客用户。

## 三、后端 `.env`

复制 `backend/.env.example` 为 `backend/.env`：

```env
# --- 基础 ---
CORS_ORIGINS=http://localhost:5173

# --- PostgreSQL，本地 Docker ---
DATABASE_URL=postgresql+psycopg://ucub:ucub_password@localhost:5432/ucub_mqdraw

# 异步 SQLAlchemy 可改为：
# DATABASE_URL=postgresql+asyncpg://ucub:ucub_password@localhost:5432/ucub_mqdraw

# --- Redis，本地 Docker ---
REDIS_URL=redis://:ucub_redis_password@localhost:6379/0

# --- WebSocket 事件总线 ---
WS_EVENT_BUS=local
WS_REDIS_CHANNEL=ucub_mqdraw:ws_events

# --- JWT ---
JWT_SECRET_KEY=please_change_me_in_production
JWT_ALGORITHM=HS256
JWT_ACCESS_TOKEN_EXPIRE_SECONDS=604800

# --- 登录限流（Phase 3）---
AUTH_LOGIN_RATE_LIMIT_IP=20
AUTH_LOGIN_RATE_LIMIT_ACCOUNT=5
AUTH_RATE_LIMIT_BACKEND=redis

# --- OSS，阿里云外部服务 ---
OSS_ACCESS_KEY_ID=
OSS_ACCESS_KEY_SECRET=
OSS_REGION=cn-guangzhou
OSS_ENDPOINT=http://object.intuly.com
OSS_USE_CNAME=true
OSS_BUCKET_NAME=
OSS_PATH=ai/img/build

# --- RocketMQ，阿里云外部服务（本地联调也必填）---
ROCKETMQ_ENABLED=true
ROCKETMQ_NAME_SERVER=
ROCKETMQ_PRODUCER_GROUP=ucub-mqdraw-producer
ROCKETMQ_CONSUMER_GROUP=ucub-mqdraw-callback-consumer
ROCKETMQ_ACCESS_KEY=
ROCKETMQ_SECRET_KEY=
ROCKETMQ_TOPIC_TASK=ucub_imggen_api_task
ROCKETMQ_TOPIC_CALLBACK=ucub_imggen_callback
ROCKETMQ_RETRY_TIMES=3
ROCKETMQ_RETRY_BACKOFF_BASE=0.5
ROCKETMQ_RETRY_BACKOFF_MAX=5.0
```

> **已废弃**：`USE_MOCK=true`（业务侧跳过 MQ + 后台线程伪造进度）。本地假生图用 `imggen-stub/`。

**密钥只放环境变量，不要提交到仓库。**

Python 依赖（Phase 1 同步 ORM）：

```bash
pip install sqlalchemy psycopg[binary] redis passlib[bcrypt] python-jose[cryptography]
```

## 四、Redis 配置说明

后端 settings 建议只保留 `REDIS_URL`：

```python
class Settings(BaseSettings):
    redis_url: str = "redis://:ucub_redis_password@localhost:6379/0"
    ws_event_bus: str = "local"          # local | redis
    ws_redis_channel: str = "ucub_mqdraw:ws_events"
```

启用 Redis Pub/Sub（Phase 3）：

```env
WS_EVENT_BUS=redis
```

消息格式（发布到 `WS_REDIS_CHANNEL`）：

```json
{
  "userId": "user_20260713_abcd1234",
  "payload": {
    "type": "task.updated",
    "eventId": "evt_20260713_xxx",
    "serverTime": "2026-07-13T00:10:00+08:00",
    "task": { "taskId": "...", "status": "running", "progress": 60 }
  }
}
```

业务代码统一走 `task_event_notifier.publish_task_event(...)`，内部根据 `WS_EVENT_BUS` 选择：

```text
local → websocket_manager.send_to_user
redis → redis.publish(WS_REDIS_CHANNEL, ...)
```

详见 [WebSocket推送机制.md §九](./WebSocket推送机制.md#九多实例部署)。

## 六、启动顺序

```text
1. docker compose up -d          # PostgreSQL + Redis
2. docker compose ps             # 确认 healthy
3. 配置 backend/.env（含 RocketMQ）
4. 启动 UcubMQDraw 后端
5. 启动 imggen-stub（或真实 imggen）
6. 启动前端
7. 浏览器访问 http://localhost:5173
```

```bash
# 1. 本地基础服务
docker compose up -d
docker compose ps

# 2. 后端
cd backend
uv sync
uv run uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

# 3. imggen-stub（假生图，走真实 MQ）
cd imggen-stub
uv run python -m app.main

# 4. 前端
cd frontend
npm install
npm run dev
```

## 七、环境对照（无 USE_MOCK）

| 环境 | UcubMQDraw MQ | imggen 服务 | PostgreSQL | Redis | OSS | RocketMQ |
|------|---------------|-------------|------------|-------|-----|----------|
| 本地开发 | 开启 | **imggen-stub** | Docker 必填 | Docker 建议启 | 可选固定 URL | **必填** |
| 联调 imggen | 开启 | 真实 imggen | Docker / 云 PG | Docker / 云 Redis | 必填 | 必填 |
| 生产 | 开启 | 真实 imggen | 云 PG / 自建 | 云 Redis | 必填 | 必填 |

本地联调：

```text
创建任务 → 投递 RocketMQ task topic
imggen-stub 消费 → sleep → 投递 callback topic
UcubMQDraw Consumer 更新 DB → WebSocket 推送
```

`imggen-stub/.env.example`：

```env
ROCKETMQ_NAME_SERVER=
ROCKETMQ_ACCESS_KEY=
ROCKETMQ_SECRET_KEY=
ROCKETMQ_CONSUMER_GROUP=imggen-stub-consumer
ROCKETMQ_PRODUCER_GROUP=imggen-stub-producer
ROCKETMQ_TOPIC_TASK=ucub_imggen_api_task
ROCKETMQ_TOPIC_CALLBACK=ucub_imggen_callback
IMGGEN_STUB_SLEEP_SECONDS=3
IMGGEN_STUB_RESULT=success
IMGGEN_STUB_OUTPUT_URL=http://object.intuly.com/ai/img/build/mock-output.png
IMGGEN_STUB_SEND_PROGRESS=false
```

详见 [imggen-stub联调模式设计.md](./imggen-stub联调模式设计.md)。

## 九、Nginx 反代（生产示例）

```nginx
# 前端
location / {
    root /var/www/ucub-mqdraw/dist;
    try_files $uri $uri/ /index.html;
}

# 后端 API
location /api/ {
    proxy_pass http://127.0.0.1:8000;
    proxy_set_header Host $host;
}

# WebSocket
location /api/v1/ws/ {
    proxy_pass http://127.0.0.1:8000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

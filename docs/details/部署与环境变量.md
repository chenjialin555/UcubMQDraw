# 部署与环境变量

## 一、后端 `.env`

```env
# --- 基础 ---
USE_MOCK=true
DATABASE_URL=sqlite:///./data/ucub_mqdraw.db
CORS_ORIGINS=http://localhost:5173

# --- JWT ---
JWT_SECRET_KEY=please_change_me_in_production
JWT_ALGORITHM=HS256
JWT_ACCESS_TOKEN_EXPIRE_SECONDS=604800

# --- 登录限流（可选，第一版可内存实现）---
AUTH_LOGIN_RATE_LIMIT_IP=20
AUTH_LOGIN_RATE_LIMIT_ACCOUNT=5

# --- OSS（开发与生产均必填） ---
OSS_ACCESS_KEY_ID=
OSS_ACCESS_KEY_SECRET=
OSS_REGION=cn-guangzhou
OSS_ENDPOINT=http://object.intuly.com
OSS_USE_CNAME=true
OSS_BUCKET_NAME=
OSS_PATH=ai/img/build

# --- RocketMQ（联调/生产） ---
ROCKETMQ_NAME_SERVER=
ROCKETMQ_PRODUCER_GROUP=ucub-mqdraw-producer
ROCKETMQ_ACCESS_KEY=
ROCKETMQ_SECRET_KEY=
ROCKETMQ_TOPIC_TASK=ucub_imggen_api_task
ROCKETMQ_TOPIC_CALLBACK=ucub_imggen_callback
ROCKETMQ_RETRY_TIMES=3
ROCKETMQ_RETRY_BACKOFF_BASE=0.5
ROCKETMQ_RETRY_BACKOFF_MAX=5.0
```

**密钥只放环境变量，不要提交到仓库。**

Python 额外依赖（鉴权）：

```bash
pip install passlib[bcrypt] python-jose[cryptography]
```

## 二、前端环境变量

### `.env.development`

```env
VITE_API_BASE_URL=http://localhost:8000
VITE_WS_URL=ws://localhost:8000/api/v1/ws/tasks
```

### `.env.production`

```env
VITE_API_BASE_URL=https://api.your-domain.com
VITE_WS_URL=wss://api.your-domain.com/api/v1/ws/tasks
```

## 三、启动命令

```bash
# 后端
cd backend
uv sync
USE_MOCK=true uv run uvicorn app.main:app --host 0.0.0.0 --port 8000

# 前端
cd frontend
npm install
npm run build    # 生产
npm run dev      # 开发
```

## 四、SQLite 路径

```text
DATABASE_URL=sqlite:///./data/ucub_mqdraw.db
```

确保 `data/` 目录存在且可写。Docker 部署时挂载 volume。

## 五、CORS

```python
# config.py
CORS_ORIGINS: str = "http://localhost:5173"

# main.py
app.add_middleware(
    CORSMiddleware,
    allow_origins=settings.cors_origins_list,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

生产改为实际前端域名。

## 六、Nginx 反代（示例）

```nginx
# 前端
location / {
    root /var/www/ucub-mqdraw/dist;
    try_files $uri $uri/ /index.html;
}

# 后端 API
location /api/ {
    proxy_pass http://127.0.0.1:8000;
    proxy_set_header Host $host;
}

# WebSocket
location /api/v1/ws/ {
    proxy_pass http://127.0.0.1:8000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

## 七、环境对照

| 环境 | USE_MOCK | OSS | MQ |
|------|----------|-----|-----|
| 本地开发 | true | 必填 | 不启 |
| 联调 imggen | false | 必填 | 启 |
| 生产 | false | 必填 | 启 |

# 数据库存储设计

## 一、为什么用 SQLite

第一版使用单文件 SQLite（`data/ucub_mqdraw.db`）：

- 开发与生产配置一致，无需额外装 MySQL
- 任务量不大时性能足够
- 后续可迁移 PostgreSQL（ORM 层不变，主要改 `DATABASE_URL`）

## 二、设计原则：固定列 + 三个 JSON 列

每种工具的参数不同（风格迁移要两张图、局部重绘要蒙版、文生图可能只要 prompt），**不适合为每种参数建扁平列**。

| 阶段 | DB 列名 | API 字段 | 存什么 | 谁写 | 谁读 |
|------|---------|----------|--------|------|------|
| 用户提交 | `task_submit_params` | `taskSubmitParams` | 前端 `params` 原文 | 创建任务 | 重跑、右栏输入预览 |
| 投递算力 | `task_dispatch_params` | —（不返前端） | Handler 组装后投 MQ 的 JSON | Handler | 排错、对账 |
| 算力回调 | `task_callback_params` | `taskCallbackParams` | imggen 回调完整 JSON | 回调 Consumer | 右栏输出预览 |

**不再单独存：**

```text
prompt / ref_image_list / mask_image_url ...  → 并入上述 JSON
input_preview / output_preview  → 前端从 JSON 解析
task_dispatch_params 字段对齐 mq_contract.md（无 workflow）
```

## 三、建表 SQL

```sql
CREATE TABLE IF NOT EXISTS generation_tasks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id TEXT NOT NULL UNIQUE,
    inner_task_id TEXT,

    user_id TEXT,
    tool_key TEXT NOT NULL,
    tool_name TEXT NOT NULL,

    status TEXT NOT NULL,
    progress INTEGER NOT NULL DEFAULT 0,

    task_submit_params TEXT NOT NULL DEFAULT '{}',
    task_dispatch_params TEXT NOT NULL DEFAULT '{}',
    task_callback_params TEXT,

    error_message TEXT,
    cost_time REAL,
    favorite INTEGER NOT NULL DEFAULT 0,

    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_tasks_status ON generation_tasks(status);
CREATE INDEX IF NOT EXISTS idx_tasks_created_at ON generation_tasks(created_at DESC);
CREATE INDEX IF NOT EXISTS idx_tasks_inner_task_id ON generation_tasks(inner_task_id);
CREATE INDEX IF NOT EXISTS idx_tasks_tool_key ON generation_tasks(tool_key);
```

## 四、固定列说明

| 列名 | 说明 |
|------|------|
| `task_id` | 业务任务 ID，前端展示用 |
| `inner_task_id` | 投 MQ 用，imggen 回调映射 |
| `tool_key` / `tool_name` | 工具标识与中文名 |
| `status` / `progress` | 任务状态与进度 0~100 |
| `error_message` | 失败原因 |
| `cost_time` | 耗时（秒） |
| `favorite` | 收藏标记，MVP 不实现 UI |

## 六、ORM 响应转换

`to_response_dict()` 将 snake_case 转 camelCase，**不返回** `task_dispatch_params`：

```python
def to_response_dict(self) -> dict:
    return {
        "taskId": self.task_id,
        "toolKey": self.tool_key,
        "toolName": self.tool_name,
        "status": self.status,
        "progress": self.progress,
        "taskSubmitParams": self.task_submit_params or {},
        "taskCallbackParams": self.task_callback_params or {},
        "errorMessage": self.error_message or "",
        "costTime": self.cost_time,
        "favorite": self.favorite,
        "createdAt": self.created_at,
        "updatedAt": self.updated_at,
    }
```

完整 ORM 示例见 [archive/完整版方案归档.md](../archive/完整版方案归档.md) §10.3。

## 七、PostgreSQL 迁移（第二版）

```text
1. 改 DATABASE_URL 为 postgresql://...

3. 索引策略不变
4. 用 Alembic 管理迁移（第一版可 create_all 自动建表）
```

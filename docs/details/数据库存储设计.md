# 数据库存储设计

## 一、数据库选型

| 环境 | 引擎 | 说明 |
|------|------|------|
| 开发 | SQLite | 单文件 `data/ucub_mqdraw.db`，快速本地联调 |
| 生产 | MySQL | 正式部署，`DATABASE_URL=mysql+pymysql://...` |

ORM 层保持一致，切换主要改 `DATABASE_URL` 与少量方言差异。

**任务队列不在数据库里**，异步执行走 RocketMQ。

---

## 二、设计原则：固定列 + 三个 JSON 列

| 阶段 | DB 列名 | API 字段 | 存什么 |
|------|---------|----------|--------|
| 用户提交 | `task_submit_params` | `taskSubmitParams` | 前端 `params` 原文 |
| 投递算力 | `task_dispatch_params` | —（不返前端） | Handler 组装后投 MQ 的 JSON |
| 算力回调 | `task_callback_params` | `taskCallbackParams` | imggen 回调完整 JSON |

不再单独存 `prompt` / `ref_image_list` / `workflow` 等扁平列。

## 三、users 表

```sql
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    user_id VARCHAR(64) NOT NULL,
    user_type VARCHAR(20) NOT NULL,

    phone VARCHAR(32) NULL,
    email VARCHAR(128) NULL,

    phone_verified TINYINT(1) NOT NULL DEFAULT 0,
    email_verified TINYINT(1) NOT NULL DEFAULT 0,

    password_hash VARCHAR(255) NULL,

    nickname VARCHAR(64) NOT NULL,
    avatar_url VARCHAR(512) NULL,

    status VARCHAR(20) NOT NULL DEFAULT 'active',

    last_login_at DATETIME NULL,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,

    UNIQUE KEY uniq_user_id (user_id),
    UNIQUE KEY uniq_phone (phone),
    UNIQUE KEY uniq_email (email),
    KEY idx_user_type (user_type),
    KEY idx_created_at (created_at)
);
```

| 字段 | 说明 |
|------|------|
| `user_id` | 业务 ID，如 `user_20260712_ab12cd34`；游客固定 `user_guest_public` |
| `user_type` | `phone` / `email` / `guest` |
| `phone` / `email` | 注册账号，互斥填写 |
| `phone_verified` / `email_verified` | 第一版默认 `0`，后续接验证码 |
| `password_hash` | bcrypt；游客为 NULL |
| `nickname` | 默认随机「用户XXXXXX」 |
| `status` | `active` / `disabled` |

游客初始化见 [用户与鉴权设计.md §七](./用户与鉴权设计.md#七数据库)。

### 可选：refresh_tokens 表

第一版可不做。若需 refreshToken：

```sql
CREATE TABLE refresh_tokens (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id VARCHAR(64) NOT NULL,
    token_hash VARCHAR(255) NOT NULL,
    expires_at DATETIME NOT NULL,
    revoked_at DATETIME NULL,
    created_at DATETIME NOT NULL,
    KEY idx_user_id (user_id),
    KEY idx_expires_at (expires_at)
);
```

## 四、generation_tasks 表

```sql
CREATE TABLE generation_tasks (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    task_id VARCHAR(64) NOT NULL,
    inner_task_id VARCHAR(64) NOT NULL,
    user_id VARCHAR(64) NOT NULL,

    tool_key VARCHAR(64) NOT NULL,
    tool_name VARCHAR(128) NOT NULL,

    status VARCHAR(32) NOT NULL,
    progress INT NOT NULL DEFAULT 0,

    task_submit_params JSON NOT NULL,
    task_dispatch_params JSON NOT NULL,
    task_callback_params JSON NULL,

    error_message TEXT NULL,
    cost_time DOUBLE NULL,
    favorite TINYINT(1) NOT NULL DEFAULT 0,

    deleted_at DATETIME NULL,
    started_at DATETIME NULL,
    finished_at DATETIME NULL,
    retry_count INT NOT NULL DEFAULT 0,

    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,

    UNIQUE KEY uniq_task_id (task_id),
    UNIQUE KEY uniq_inner_task_id (inner_task_id),
    KEY idx_user_created (user_id, created_at DESC),
    KEY idx_user_status (user_id, status),
    KEY idx_status_updated (status, updated_at)
);
```

### SQLite 开发库

字段与上一致；JSON 列可用 `TEXT`，时间用 `TEXT` ISO 字符串。索引同名创建。

## 五、固定列说明（任务表）

| 列名 | 说明 |
|------|------|
| `task_id` | 业务任务 ID，前端展示，`ucub_task_YYYYMMDD_xxxxxxxxxx` |
| `inner_task_id` | MQ / imggen 映射，`inner_YYYYMMDD_xxxxxxxxxx` |
| `user_id` | **必填**，权限隔离 + WS 定向推送 |
| `tool_key` / `tool_name` | 工具标识与中文名 |
| `status` | 见 [API接口.md §十一](../API接口.md#十一任务状态枚举) |
| `progress` | 0~100 |
| `error_message` | 失败原因 |
| `cost_time` | 耗时（秒） |
| `favorite` | 收藏 |
| `deleted_at` | 软删除时间，NULL=未删除 |
| `started_at` | 进入 running 的时间 |
| `finished_at` | 进入终态的时间 |
| `retry_count` | 重试计数（可选） |

正式版**必须有 `user_id`**，否则无法做用户级 WebSocket 推送与权限校验。

## 六、ID 生成规则

```python
def generate_task_id() -> str:
    date_part = datetime.now().strftime("%Y%m%d")
    random_part = uuid4().hex[:10]
    return f"ucub_task_{date_part}_{random_part}"


def generate_inner_task_id() -> str:
    date_part = datetime.now().strftime("%Y%m%d")
    random_part = uuid4().hex[:10]
    return f"inner_{date_part}_{random_part}"
```

- `taskId` 暴露给前端
- `innerTaskId` 仅 DB + MQ，前端不展示、不生成

## 七、Repository 职责

### user_repo

```text
create(...)
get_by_user_id(user_id)
get_by_phone(phone)
get_by_email(email)
update_last_login(user_id)
update_nickname(user_id, nickname)
update_password_hash(user_id, password_hash)
```

### task_repo

```text
create(user_id, tool_key, ...)
list_by_user(user_id, page, page_size, status?)
get_by_task_id(task_id)
get_by_inner_task_id(inner_task_id)
update_status(task_id, status, progress)
mark_succeeded(task_id, task_callback_params, cost_time)
mark_failed(task_id, error_message, task_callback_params?)
update_progress(...)
soft_delete(task_id)          → 设置 deleted_at
update_favorite(task_id, favorite)
```

查询列表默认 `WHERE deleted_at IS NULL AND user_id = ?`。

## 八、Pydantic Schema（摘要）

见 [API接口.md §十二](../API接口.md#十二前端-typescript-类型参考) 与后端：

```python
class CreateTaskRequest(BaseModel):
    toolKey: str
    params: dict[str, Any] = Field(default_factory=dict)

class TaskResponse(BaseModel):
    taskId: str
    toolKey: str
    toolName: str
    status: str
    progress: int
    taskSubmitParams: dict[str, Any] = Field(default_factory=dict)
    taskCallbackParams: dict[str, Any] = Field(default_factory=dict)
    errorMessage: str = ""
    costTime: Optional[float] = None
    favorite: bool = False
    createdAt: str
    updatedAt: str

class FavoriteTaskRequest(BaseModel):
    favorite: bool
```

`to_response_dict()` **不返回** `task_dispatch_params`、`user_id`、`inner_task_id`。

## 九、ORM 响应转换

```python
def to_response_dict(self) -> dict:
    return {
        "taskId": self.task_id,
        "toolKey": self.tool_key,
        "toolName": self.tool_name,
        "status": self.status,
        "progress": self.progress,
        "taskSubmitParams": self.task_submit_params or {},
        "taskCallbackParams": self.task_callback_params or {},
        "errorMessage": self.error_message or "",
        "costTime": self.cost_time,
        "favorite": bool(self.favorite),
        "createdAt": self.created_at,
        "updatedAt": self.updated_at,
    }
```

## 十、与归档的差异

| 项 | 旧文档 | 现行 |
|----|--------|------|
| 终态名 | `completed` | `succeeded` |
| 取消 | `cancelled` | `canceled` |
| 用户表 | 无 | `users` 表 + JWT |
| 软删除 | 无 | `deleted_at` |
| 生产库 | SQLite | MySQL |

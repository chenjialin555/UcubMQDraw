## 一、数据库选型

| 环境 | 引擎 | 说明 |
|------|------|------|
| 开发 | PostgreSQL | Docker Compose 启动，本地统一使用 PG |
| 生产 | PostgreSQL | 云数据库 RDS PostgreSQL 或自建 PG |

项目**不再使用 SQLite / MySQL**。开发、联调、生产统一 PostgreSQL，避免 JSON、索引、时间类型在不同数据库之间产生差异。

**任务队列不在数据库里**，异步执行走阿里云 **RocketMQ**（外部服务，不在 Docker 中启动）。

### 迁移策略

| 阶段 | 方式 |
|------|------|
| Phase 1 | `docker/postgres/init/01_init.sql` 初始化表 |
| Phase 2 | 接入 Alembic migration |

Phase 1 先用 `init.sql`，简单直观；感觉代码阅读压力大时**不要**提前引入 Alembic。

## 二、设计原则：固定列 + 三个 JSONB 列

| 阶段 | DB 列名 | API 字段 | 存什么 |
|------|---------|----------|--------|
| 用户提交 | `task_submit_params` | `taskSubmitParams` | 前端 `params` 原文 |
| 投递算力 | `task_dispatch_params` | —（不返前端） | Handler 组装后投 MQ 的 JSON |
| 算力回调 | `task_callback_params` | `taskCallbackParams` | imggen 回调完整 JSON |

不再单独存 `prompt` / `ref_image_list` / `workflow` 等扁平列。PostgreSQL 使用 **JSONB** 存储，支持索引与高效查询。

## 三、users 表

```sql
CREATE TABLE IF NOT EXISTS users (
    id BIGSERIAL PRIMARY KEY,

    user_id VARCHAR(64) NOT NULL,
    user_type VARCHAR(20) NOT NULL,

    phone VARCHAR(32),
    email VARCHAR(128),

    phone_verified BOOLEAN NOT NULL DEFAULT FALSE,
    email_verified BOOLEAN NOT NULL DEFAULT FALSE,

    password_hash VARCHAR(255),

    nickname VARCHAR(64) NOT NULL,
    avatar_url VARCHAR(512),

    status VARCHAR(20) NOT NULL DEFAULT 'active',

    last_login_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE UNIQUE INDEX IF NOT EXISTS uniq_users_user_id ON users(user_id);

CREATE UNIQUE INDEX IF NOT EXISTS uniq_users_phone ON users(phone)
WHERE phone IS NOT NULL;

CREATE UNIQUE INDEX IF NOT EXISTS uniq_users_email ON users(email)
WHERE email IS NOT NULL;
```

| 字段 | 说明 |
|------|------|
| `user_id` | 业务 ID，如 `user_20260712_ab12cd34`；游客固定 `user_guest_public` |
| `user_type` | `phone` / `email` / `guest` |
| `phone` / `email` | 注册账号，互斥填写 |
| `phone_verified` / `email_verified` | 第一版默认 `false`，后续接验证码 |
| `password_hash` | bcrypt；游客为 NULL |
| `nickname` | 默认随机「用户XXXXXX」 |
| `status` | `active` / `disabled` |

`WHERE phone IS NOT NULL` 为 PostgreSQL **部分唯一索引**，允许多个 NULL，同时保证非空手机号唯一。

游客初始化见 `docker/postgres/init/01_init.sql` 末尾 `INSERT`，以及 [用户与鉴权设计.md §九](./用户与鉴权设计.md#九数据库)。

## 四、generation_tasks 表

```sql
CREATE TABLE IF NOT EXISTS generation_tasks (
    id BIGSERIAL PRIMARY KEY,

    task_id VARCHAR(64) NOT NULL,
    inner_task_id VARCHAR(64) NOT NULL,
    user_id VARCHAR(64) NOT NULL,

    tool_key VARCHAR(64) NOT NULL,
    tool_name VARCHAR(128) NOT NULL,

    status VARCHAR(32) NOT NULL,
    progress INTEGER NOT NULL DEFAULT 0,

    task_submit_params JSONB NOT NULL DEFAULT '{}'::jsonb,
    task_dispatch_params JSONB NOT NULL DEFAULT '{}'::jsonb,
    task_callback_params JSONB,

    error_message TEXT,
    cost_time DOUBLE PRECISION,
    favorite BOOLEAN NOT NULL DEFAULT FALSE,

    deleted_at TIMESTAMPTZ,
    started_at TIMESTAMPTZ,
    finished_at TIMESTAMPTZ,
    retry_count INTEGER NOT NULL DEFAULT 0,

    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

索引见 `01_init.sql`。时间字段统一 **TIMESTAMPTZ**（带时区）。

## 五、固定列说明（任务表）

| 列名 | 说明 |
|------|------|
| `task_id` | 业务任务 ID，前端展示，`ucub_task_YYYYMMDD_xxxxxxxxxx` |
| `inner_task_id` | MQ / imggen 映射，`inner_YYYYMMDD_xxxxxxxxxx` |
| `user_id` | **必填**，权限隔离 + WS 定向推送 |
| `tool_key` / `tool_name` | 工具标识与中文名 |
| `status` | 见 [API接口.md §十二](../API接口.md#十二任务状态枚举) |
| `progress` | 0~100 |
| `error_message` | 失败原因 |
| `cost_time` | 耗时（秒） |
| `favorite` | 收藏 |
| `deleted_at` | 软删除时间，NULL=未删除 |
| `started_at` | 进入 running 的时间 |
| `finished_at` | 进入终态的时间 |
| `retry_count` | 重试计数（可选） |

正式版**必须有 `user_id`**，否则无法做用户级 WebSocket 推送与权限校验。

## 六、ID 生成规则

```python
def generate_task_id() -> str:
    date_part = datetime.now().strftime("%Y%m%d")
    random_part = uuid4().hex[:10]
    return f"ucub_task_{date_part}_{random_part}"


def generate_inner_task_id() -> str:
    date_part = datetime.now().strftime("%Y%m%d")
    random_part = uuid4().hex[:10]
    return f"inner_{date_part}_{random_part}"
```

- `taskId` 暴露给前端
- `innerTaskId` 仅 DB + MQ，前端不展示、不生成

## 九、ORM 与 JSONB

PostgreSQL 推荐：

```python
from sqlalchemy import BigInteger, Boolean, Column, DateTime, Float, Integer, String, Text
from sqlalchemy.dialects.postgresql import JSONB

class GenerationTask(Base):
    __tablename__ = "generation_tasks"

    id = Column(BigInteger, primary_key=True, autoincrement=True)
    task_id = Column(String(64), nullable=False, unique=True)
    inner_task_id = Column(String(64), nullable=False, unique=True)
    user_id = Column(String(64), nullable=False)

    tool_key = Column(String(64), nullable=False)
    tool_name = Column(String(128), nullable=False)

    status = Column(String(32), nullable=False)
    progress = Column(Integer, nullable=False, default=0)

    task_submit_params = Column(JSONB, nullable=False, default=dict)
    task_dispatch_params = Column(JSONB, nullable=False, default=dict)
    task_callback_params = Column(JSONB, nullable=True)

    error_message = Column(Text, nullable=True)
    cost_time = Column(Float, nullable=True)
    favorite = Column(Boolean, nullable=False, default=False)
```

数据库连接（同步 SQLAlchemy + psycopg）：

```python
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker

engine = create_engine(
    settings.DATABASE_URL,
    pool_pre_ping=True,
    pool_size=5,
    max_overflow=10,
)
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)
```

## 十、响应转换

```python
def to_response_dict(self) -> dict:
    return {
        "taskId": self.task_id,
        "toolKey": self.tool_key,
        "toolName": self.tool_name,
        "status": self.status,
        "progress": self.progress,
        "taskSubmitParams": self.task_submit_params or {},
        "taskCallbackParams": self.task_callback_params or {},
        "errorMessage": self.error_message or "",
        "costTime": self.cost_time,
        "favorite": bool(self.favorite),
        "createdAt": self.created_at,
        "updatedAt": self.updated_at,
    }
```

## 十一、与归档的差异

| 项 | 旧文档 | 现行 |
|----|--------|------|
| 终态名 | `completed` | `succeeded` |
| 取消 | `cancelled` | `canceled` |
| 用户表 | 无 | `users` 表 + JWT |
| 软删除 | 无 | `deleted_at` |
| 开发库 | SQLite | **PostgreSQL Docker** |
| 生产库 | MySQL | **PostgreSQL** |
| JSON 列 | `JSON` / `TEXT` | **JSONB** |
| 时间列 | `DATETIME` | **TIMESTAMPTZ** |

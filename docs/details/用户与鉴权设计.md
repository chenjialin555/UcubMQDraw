# 用户与鉴权设计

> 轻量账号密码体系：手机号 / 邮箱注册登录、JWT、游客模式、任务 `user_id` 隔离。
> HTTP 契约见 [API接口.md](../API接口.md#二认证接口)；WS 见 [WebSocket推送机制.md](./WebSocket推送机制.md)。
>
> **先读一句：** 登录得到 token，后端从 token 知道你是谁；任务写 `user_id`；WS 只推你的任务。
> 阅读顺序 → [代码阅读指南.md](../代码阅读指南.md)

## 一、一句话核心

```text
注册 / 登录 / 游客
    ↓
拿到 accessToken
    ↓
HTTP 请求带 Authorization: Bearer token
    ↓
后端解析 user_id
    ↓
创建任务写入 generation_tasks.user_id
    ↓
WebSocket send_to_user(user_id) 推送
```

JWT、bcrypt、游客过滤、拦截器——都围绕上面这条链。

## 三、定位

```text
轻量账号系统
```

**Phase 1 必做：**

```text
注册 / 登录 / 游客 / me
JWT 鉴权
任务按 user_id 隔离

```

**Phase 2 再做：**

```text
修改昵称 / 修改密码 / 退出登录接口

```

**不做（或 Phase 3）：**

```text
短信 / 邮箱验证码
找回密码
第三方登录
角色权限 / 组织团队
实名认证
refreshToken（Phase 3）
登录限流（Phase 3）
```

> 不做验证码 = 只能证明「用户填了这个号/邮箱」，不能证明归属。`phone_verified` / `email_verified` 默认 `false`。

|----------|------|
| `phone` | 手机号 + 密码注册 |
| `email` | 邮箱 + 密码注册 |

## 五、整体架构

```text
Vue 前端
  ↓ 注册 / 登录 HTTP
FastAPI Auth API
  ↓ users 表（bcrypt 密码）
  ↓ 返回 JWT accessToken
  ↓
前端 localStorage 保存 token
  ↓ Authorization: Bearer <token>
  ↓
创建任务 → generation_tasks.user_id = token.sub
  ↓
imggen 回调 → send_to_user(task.user_id)
  ↓
WebSocket ?token=<jwt> → 解析 user_id → 定向推送
```

## 六、注册 / 登录规则

### 4.1 account 格式

| 类型 | 规则 |
|------|------|
| 手机号 | `^1[3-9]\d{9}$` |
| 邮箱 | `^[^\s@]+@[^\s@]+\.[^\s@]+$` |

```python
def detect_account_type(account: str) -> str:
    account = account.strip()
    if re.match(r"^1[3-9]\d{9}$", account):
        return "phone"
    if re.match(r"^[^\s@]+@[^\s@]+\.[^\s@]+$", account):
        return "email"
    raise BizError("INVALID_ACCOUNT", "请输入正确的手机号或邮箱")
```

### 4.2 密码规则

```text
长度 8 ~ 32 位
不能全是空格
第一版不强制复杂度
```

存储：**bcrypt**，绝不存明文。

```python
from passlib.context import CryptContext

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

def hash_password(password: str) -> str:
    return pwd_context.hash(password)

def verify_password(password: str, password_hash: str) -> bool:
    return pwd_context.verify(password, password_hash)
```

依赖：`passlib[bcrypt]`、`python-jose`。

## 七、JWT

### 5.1 Payload

```json
{
  "sub": "user_20260712_ab12cd34",
  "userType": "phone",
  "exp": 1783870000
}
```

| 字段 | 说明 |
|------|------|
| `sub` | 业务 `user_id` |
| `userType` | `phone` / `email` / `guest` |
| `exp` | 过期时间 |

### 5.2 环境变量

```env
JWT_SECRET_KEY=please_change_me
JWT_ALGORITHM=HS256
JWT_ACCESS_TOKEN_EXPIRE_SECONDS=604800
```

第一版 accessToken 有效期 **7 天**（604800 秒）。退出登录前端删 token 即可；第一版不做 token 黑名单。

## 十一、错误码

统一格式：

```json
{
  "code": "ACCOUNT_EXISTS",
  "message": "账号已注册"
}
```

| code | message |
|------|---------|
| `INVALID_ACCOUNT` | 请输入正确的手机号或邮箱 |
| `PASSWORD_TOO_SHORT` | 密码至少 8 位 |
| `ACCOUNT_EXISTS` | 账号已注册 |
| `INVALID_LOGIN` | 账号或密码错误（登录统一文案，不暴露账号是否存在） |
| `USER_DISABLED` | 用户已被禁用 |
| `UNAUTHORIZED` | 请先登录 |
| `TOKEN_EXPIRED` | 登录已过期 |
| `GUEST_NOT_ALLOWED` | 游客不允许执行该操作 |

任务权限：操作他人任务返回 **404**（不返回「无权限」）。

## 十三、核心逻辑

### 11.1 注册

```text
account + password → detect_account_type
→ 校验密码长度 → 查重 phone/email
→ generate_user_id + nickname + hash_password
→ 写 users → create_access_token → 返回
```

### 11.2 登录

```text
account → 查 user → 校验 status + verify_password
→ update_last_login → create_access_token
```

### 11.3 HTTP 鉴权依赖

```python
async def get_current_user(authorization: str = Header(None)) -> User:
    token = extract_bearer_token(authorization)
    payload = decode_token(token)
    user = user_repo.get_by_user_id(payload["sub"])
    if not user or user.status != "active":
        raise HTTPException(401, detail={"code": "UNAUTHORIZED", "message": "请先登录"})
    return user
```

### 11.4 WebSocket 鉴权

```text
ws://host/api/v1/ws/tasks?token=<jwt>
```

```python
async def get_current_user_from_ws(websocket: WebSocket, token: str | None) -> User:
    if not token:
        await websocket.close(code=1008)
        raise WebSocketException(code=1008)
    payload = decode_token(token)
    user = user_repo.get_by_user_id(payload["sub"])
    if not user:
        await websocket.close(code=1008)
        raise WebSocketException(code=1008)
    return user
```

**禁止** `/ws/connect/{user_id}`——身份必须从 token 解析。

## 十四、任务与用户隔离

```python
async def create_task(current_user: User, req: CreateTaskRequest):
    task = task_repo.create(user_id=current_user.user_id, ...)

async def list_tasks(current_user: User, page, page_size):
    return task_repo.list_by_user(current_user.user_id, page, page_size)

async def delete_task(current_user: User, task_id: str):
    task = task_repo.get_by_task_id(task_id)
    if not task or task.user_id != current_user.user_id:
        raise HTTPException(404)
```

## 十五、游客模式（Phase 2 增强过滤）

### 13.1 规则

| 操作 | 游客 |
|------|------|
| 创建任务 | ✅ |
| 查看服务端全量历史 | ❌（共用 `user_guest_public`） |
| 查看当前浏览器任务 | ✅（前端本地 taskId 过滤） |
| 删除 / 收藏 / 改昵称 / 改密码 | ❌ |

### 13.2 产品流程

```text
未登录 → bootstrap 调 POST /auth/guest → 拿 guest token
想保存个人历史 → 注册 / 登录 → 换正式 token → fetchTasks
```

### 13.3 前端本地过滤

```ts
// localStorage: guest_task_ids
function onWsTask(message: WsTaskEvent) {
  if (authStore.isGuest && message.task) {
    if (!guestTaskIds.includes(message.task.taskId)) return
  }
  taskStore.upsertTask(message.task)
}
```

创建任务成功后：

```ts
if (authStore.isGuest) {
  guestTaskIds.add(task.taskId)
  saveGuestTaskIds()
}
```

## 十七、安全最低要求

```text
1. 密码 bcrypt hash
2. JWT_SECRET_KEY 放环境变量
3. 登录接口限流（同 IP 20/min，同账号 5/min，第一版可内存实现）
4. 登录失败统一「账号或密码错误」
5. 返回手机号/邮箱脱敏（138****8000、te***@example.com）
```

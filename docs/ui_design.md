# UcubMQDraw Gradio 前端 UI 设计规范

> 版本：v1.1 · 2026-07-11  
> 适用：Gradio 6.x · 桌面端运营工作台  
> 设计风格：**AI 创作工具台**（非后台表单）

---

## 1. 设计目标

### 1.1 要达成的观感

- 像 **AI 创作平台**，不像低代码后台或默认 Gradio 表单页。
- 三栏清晰：左配置、中预览、右任务流。
- 卡片化、图标化、短句化，减少开发向说明文案。
- 弱化 Gradio 原生组件痕迹（Radio 标签、粗黑上传框等）。

### 1.2 明确避免

| 避免项 | 原因 |
|--------|------|
| `gr.Radio` 展示工具选择 | 会出现「单选框」字样，破坏质感 |
| 默认上传区黑虚线大框 | 视觉过重，像后台附件上传 |
| 长段开发说明文案 | 如「表单已预留」「后续接入」 |
| 三栏与 Gradio `scale` 混用 | 易导致左栏挤压、比例失真 |

## 2. 整体布局

### 2.1 三栏比例

桌面端采用 CSS Grid 固定比例（非纯百分比）：

```text
┌────────────────────┬────────────────────────┬────────────────────┐
│ 左栏 · 创作配置      │ 中栏 · 素材预览          │ 右栏 · 创作记录      │
│ min 360px / 28fr   │ min 520px / 44fr       │ min 300px / 28fr   │
└────────────────────┴────────────────────────┴────────────────────┘
```

- 页面最小宽度：`1280px`，小屏横向滚动。
- 栏间距：`12px`，外.padding：`12px`。
- 各栏 `height: 100%`，左右栏 `overflow-y: auto`。

### 2.2 视觉层级

```text
页面背景 (#EEF3F9)
  └── 面板 (#F8FAFD + 圆角 18px + 轻阴影)
        └── 卡片 (#FFFFFF + 圆角 16–18px)
              └── 控件 / 内容
```

### 2.3 品牌色

| Token | 色值 | 用途 |
|-------|------|------|
| `--brand` | `#123A7A` | 主按钮、选中态、强调 |
| `--brand-soft` | `#EAF1FF` | 提示条、选中背景 |
| `--brand-border` | `#BCD2F3` | 上传虚线边框 |
| `--line` | `#E5EAF2` | 分割线、边框 |
| `--muted` | `#6B7280` | 次要说明 |

样式实现：`ucub_mqdraw/ui/styles.py`（`launch(css=...)` 注入，Gradio 6 兼容）。

## 3. 左栏 · 创作配置区

### 3.1 结构自上而下

```text
UcubMQDraw 标题
分类 tabs（横向滚动）
搜索框 + 筛选 icon
创作工具 · HTML 卡片 2×3 + 透明点击层
工具提示条（短句）
风格迁移参数卡片
  ├── 底图上传
  ├── 风格参考上传
  ├── 数量 pills
  ├── 风格强度滑块 + 数值
  ├── 补充描述
  ├── 算力 / 工作流
  └── 重置
```

### 3.2 顶部分类标签

- 全部 / 图像生成 / 图像编辑 / 图转视频 / 实用工具
- **横向滚动**，`flex: 0 0 auto`，避免右侧截断。
- 当前阶段为静态展示，后续可接筛选逻辑。

### 3.3 工具选择（核心）

**禁止**用 `gr.Radio` 直接展示工具。

**采用方案：**

1. `gr.HTML` 渲染 2 行 × 3 列工具卡片（仅名称，无图标）
2. `gr.State` 保存当前工具名
3. 透明 `gr.Button` 覆盖在卡片上接收点击
4. 点击后更新 HTML 卡片选中态 + 提示文案

```text
┌────────┬────────┬────────┐
│ 风格迁移 │ 图转视频 │ 局部重绘 │
├────────┼────────┼────────┤
│ 材质替换 │ 智能扩图 │ 线稿生成 │
└────────┴────────┴────────┘
```

实现文件：

- 卡片 HTML：`ucub_mqdraw/ui/tool_grid.py` → `render_tool_grid()`
- 面板组装：`ucub_mqdraw/ui/left_panel.py`
- 点击逻辑：`ucub_mqdraw/ui/events.py` → `select_tool()`

### 3.4 工具提示文案（运营向）

| 工具 | 文案 |
|------|------|
| 风格迁移 | 上传底图和风格参考图，即可生成同风格图片 |
| 图转视频 | 图转视频能力即将开放 |
| 局部重绘 | 局部重绘能力即将开放 |
| 材质替换 | 材质替换能力即将开放 |
| 智能扩图 | 智能扩图能力即将开放 |
| 线稿生成 | 线稿生成能力即将开放 |

### 3.5 上传区

- `gr.Image(label=None)` + 上方 HTML 字段标题（底图 / 风格参考）
- 高度：`110px`
- 样式：浅蓝白底 `#F8FBFF` + 浅蓝虚线 `#BCD2F3`
- 缩小上传图标，压缩默认 Gradio 占位文案字号

### 3.6 数量选择

**禁止**用 `gr.Radio`（易显「单选框」或字段标签过重）。

**采用方案：** 4 个 pill 按钮（1张 / 2张 / 4张 / 8张）+ `gr.State` 保存当前值。

## 4. 中栏 · 素材预览区

### 4.1 结构

```text
标题：素材预览
预览容器（虚线边框大卡片）
  ├── 空状态 HTML（无图时）
  └── gr.Image 主预览（有图时覆盖在上层）
底部三按钮：重新上传 | 裁剪图片 | 立即迁移（主按钮）
提交结果短提示
```

### 4.2 空状态（必做）

无底图时显示：

```text
🖼
上传底图后在这里预览
支持 PNG / JPG / WEBP，建议使用高清图片
```

- 空状态 `pointer-events: none`，不阻挡交互。
- 左侧上传底图后，通过 `update_preview()` 清空空状态 HTML。

### 4.3 主按钮

- 文案：**立即迁移**
- 色：`#123A7A`，圆角 13px，轻阴影
- 提交过程置灰防重复点击

---

## 5. 右栏 · 创作记录区

### 5.1 结构

```text
标题 + 筛选 icon
筛选 pills：全部 | 进行中 | 已完成 | 收藏（四等分网格）
任务卡片列表（HTML 渲染）
Timer 1s 自动刷新
```

### 5.2 空状态

```text
✨
暂无创作记录
提交任务后会自动显示生成进度
```

### 5.3 任务卡片类型

| 状态 | 视觉 |
|------|------|
| 进行中 | 蓝色 badge + 进度条 + 取消 |
| 已完成 | 绿色 badge + 输入/输出缩略图 + 收藏/下载/更多 |
| 失败 | 红色 badge + 错误短句 + 重跑 |

卡片由 `ucub_mqdraw/ui/right_panel.py` → `render_task_cards()` 输出 HTML。

## 6. 前端模块与职责

```text
ucub_mqdraw/ui/
├── layout.py        # 三栏装配 + 事件绑定
├── left_panel.py    # 左栏组件
├── center_panel.py  # 中栏预览
├── right_panel.py   # 右侧任务卡片 HTML
├── events.py        # 交互逻辑（校验、提交、筛选）
├── styles.py        # CUSTOM_CSS
└── tool_grid.py     # 工具卡片 HTML 与提示文案
```

| 模块 | 职责 |
|------|------|
| `layout.py` | `build_app()`、`bind_events()` |
| `left_panel.py` | 工具区、表单、State |
| `center_panel.py` | 预览区、空状态、提交按钮 |
| `right_panel.py` | 任务流 HTML 卡片 |
| `events.py` | 工具切换、数量选择、提交、刷新 |
| `styles.py` | 全局视觉、Grid 布局、组件覆盖 |
| `tool_grid.py` | 工具卡片渲染 |

## 7. 关键交互流程

### 7.1 风格迁移提交

```text
上传底图 + 风格图 → 中栏实时预览
填写参数 → 点击「立即迁移」
  → 校验（缺图 / 超 300 字 / 非开放工具）
  → 创建任务 → mock 或调 FastAPI
  → 右栏新增卡片，Timer 刷新进度
```

### 7.2 状态刷新

| 阶段 | 机制 |
|------|------|
| Mock 演示 | `mock_task_runner.py` + `store.py` + Timer |
| 正式联调 | WebSocket 推送 + Timer 降级 |

## 8. Gradio 6 兼容要点

- `css` / `theme` 传给 `launch()`，**不要**放在 `gr.Blocks()` 构造器。
- `show_api` 已移除，使用 `footer_links=[]`。
- `Image` 不再支持 `show_download_button` 等参数。
- 三栏布局用 **CSS Grid** 控制，`Column` 不设 `scale`。

## 9. 验收清单

### 9.1 视觉

- [ ] 三栏比例正常，左栏不挤压
- [ ] 工具区为 HTML 卡片，无「单选框」字样
- [ ] 上传区浅蓝虚线，非黑色粗框
- [ ] 中栏有空状态引导
- [ ] 右栏筛选 pills 四等分不挤扁
- [ ] 主按钮深蓝，卡片白底轻阴影

### 9.2 交互

- [ ] 工具卡片可点击切换，选中态明显
- [ ] 数量 pill 可切换，默认 2 张
- [ ] 上传底图后中栏预览，空状态消失
- [ ] 提交后右栏出现任务，进度自动刷新
- [ ] 未开放工具提交时提示「暂未开放」

### 9.3 文案

- [ ] 无「表单已预留」「后续接入」等开发用语
- [ ] 提示条为短句，面向运营人员

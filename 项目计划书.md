# UcubMQDraw 优卡普AI生图异步调度平台 完整版技术方案
## 一、项目基础信息
1. **项目代号**：UcubMQDraw
2. **中文全称**：优卡普MQ异步AI生图运营调度中台
3. **建设主体**：UcubMQDraw（Python+Gradio+FastAPI 业务中台）
4. **底层算力微服务**：imggen（基于ComfyUI工作流服务化改造，分双版本）
5. **中间件**：阿里云RocketMQ 按量付费标准版（内网VPC通信）
6. **使用人群**：公司内部30人运营团队
7. **核心隔离规则**
    - imggen 仅负责图像生成算力逻辑，**不接收、不存储、不感知任何用户/运营业务信息**；
    - 用户ID、操作人、业务统计、权限、素材归属等全部数据仅留存于 UcubMQDraw 业务中台；
    - 显卡服务器无公网端口暴露，仅内网消费MQ消息，保障算力安全。

## 二、项目建设背景
1. 现有底层算力服务 imggen 基于 ComfyUI 工作流封装，支持将ComfyUI导出的工作流JSON自动解析、转换为可执行推理代码，在ComfyUI节点环境运行绘图任务；
2. imggen 拆分两个独立版本，性能与并发能力差异巨大，需要上层调度平台统一管控任务分发：
    - imggen-ComfyUI-API版：无显卡占用限制，支持多并发、多工作流队列并行执行；
    - imggen-ComfyUI-GPT版：依赖显卡资源，单实例同一时间仅能串行执行1个绘图任务；
3. 显卡算力机器无法开放公网，不能直接对外提供HTTP绘图接口；
4. 内部运营人员需要可视化操作页面、任务管理、素材库、数据统计、失败重跑能力；
5. 采用阿里云RocketMQ实现业务中台与算力微服务异步解耦，削峰排队、任务状态回调，同时搭建轻量化数据中台沉淀全部运营绘图数据。

## 三、项目建设目标
1. **算力安全隔离**：显卡服务仅内网消费MQ，杜绝公网暴露风险；
2. **多版本算力统一调度**：兼容imggen API并发版、GPT串行版两种算力实例，平台自动分发任务；
3. **业务与算力解耦**：底层imggen纯算力无业务逻辑，用户、运营数据完全隔离在UcubMQDraw；
4. **可视化低门槛运营**：Gradio可视化页面，运营人员无需对接接口即可完成文生图、图生图批量操作；
5. **标准化对外接口**：对标火山Seedance视频生成API规范，提供统一REST生图任务接口；
6. **全链路数据中台**：持久化任务、用户、素材、提示词数据，提供运营看板、任务检索导出；
7. **低成本消息调度**：阿里云RocketMQ小流量免费额度覆盖业务，月度调度成本极低；
8. **任务容错能力**：MQ消息重试、失败任务一键重跑、算力异常任务状态回写。

## 四、整体系统分层架构
自上而下五层分层：可视化运营层 → UcubMQDraw业务调度中台层 → MQ消息调度层 → 双版本imggen算力推理层 → 统一存储层
### 4.1 可视化运营层（集成在UcubMQDraw）
面向30名内部运营人员，Gradio可视化页面能力：
1. 绘图工作台：提示词/反向提示词、画布尺寸、批量数量、参考图上传、算力版本选择（API并发/GPT串行）、ComfyUI工作流模板选择；
2. 任务管理面板：分页展示所有任务，筛选排队/生成中/成功/失败，支持单任务/批量重跑、删除；
3. 素材资源库：历史生成图、运营上传参考图、ComfyUI工作流模板分类收藏；
4. 运营数据看板：日/月度作图总量、人均生成量、任务失败率、两个算力版本调用占比；
5. 账号权限体系：普通运营仅查看本人任务，管理员查看全量数据、导出报表；
6. WebSocket实时推送：任务完成自动刷新图片预览。

### 4.2 UcubMQDraw 业务调度中台层（项目核心）
内置FastAPI接口服务、MQ生产/消费客户端、数据中台、ID映射管理模块，承载全部业务逻辑。
#### 4.2.1 标准化对外REST API（对齐Seedance规范）
1. 创建生图任务
`POST /api/v1/images/generations/tasks`
请求体（完整携带业务用户信息，上层前端无感知）
```json
{
  "taskId": "ucub_task_20260711_00001",
  "userId": "op021",
  "modelName": "SDXL/Flux",
  "prompt": "高清商品图，8K，写实",
  "negativePrompt": "模糊，水印，低画质",
  "width": 1024,
  "height": 1024,
  "batchSize": 2,
  "refImageList": ["oss://ucub-img/ref/xxx.jpg"],
  "timeout": 300,
  "computeType": "comfy_api", // 可选 comfy_api / comfy_gpt
  "workflowJson": "ComfyUI导出工作流JSON字符串"
}
```
2. 查询单个任务状态
`GET /api/v1/images/generations/tasks/{taskId}`
3. 失败任务重生成
`POST /api/v1/images/generations/tasks/{taskId}/retry`

#### 4.2.2 内部ID映射核心模块（业务/算力数据隔离关键）
1. 运营提交任务后，平台生成两类唯一ID：
   - 业务taskId：绑定userId、操作人员、素材、运营标签，存入MySQL业务主表；
   - 算力innerTaskId：纯算力标识，仅用于和imggen通信，不含任何用户信息；
2. 数据库映射表 `task_inner_mapping`
| 字段 | 说明 |
| ---- | ---- |
| taskId | 平台业务任务ID，关联全量运营数据 |
| innerTaskId | 下发至imggen的纯算力ID |
| computeType | 算力类型 comfy_api / comfy_gpt |

3. 数据隔离逻辑：
平台接收上层完整带用户的参数后，剥离userId、账号、运营标签等业务字段，仅保留纯绘图参数+innerTaskId组装MQ消息下发imggen；imggen回调结果后，平台通过innerTaskId反向匹配taskId，更新业务任务状态。

#### 4.2.3 MQ生产者模块
根据请求传入的 `computeType` 分发至不同任务队列，实现两种算力服务流量隔离：
- comfy_api任务队列：`ucub_imggen_api_task`
- comfy_gpt任务队列：`ucub_imggen_gpt_task`

下发给imggen的消息体（纯净绘图参数，无任何用户业务字段）
```json
{
  "innerTaskId": "inner_20260711_00001",
  "modelName": "SDXL/Flux",
  "prompt": "高清商品图，8K，写实",
  "negativePrompt": "模糊，水印，低画质",
  "width": 1024,
  "height": 1024,
  "batchSize": 2,
  "refImageList": ["oss://ucub-img/ref/xxx.jpg"],
  "timeout": 300,
  "workflowJson": "ComfyUI原始工作流JSON"
}
```

#### 4.2.4 MQ回调消费者模块
监听统一回调队列 `ucub_imggen_callback`，接收imggen返回的纯算力结果消息：
```json
{
  "innerTaskId": "inner_20260711_00001",
  "status": "success", // success / fail
  "costTime": 18.6,
  "imageUrlList": ["oss://ucub-img/output/xxx.png"],
  "errorMsg": ""
}
```
消费逻辑：通过innerTaskId查询映射表，匹配业务taskId，更新MySQL任务状态、图片地址、耗时、错误日志，同步更新Gradio前端看板数据。

#### 4.2.5 内置轻量化数据中台
1. 核心数据表：用户表、生图任务主表、task_inner_mapping映射表、素材资源表、ComfyUI工作流模板表、每日运营统计表；
2. 数据能力：
    - 全量存储用户操作、绘图参数、生成成品地址；
    - APScheduler定时聚合每日作图数据，为Gradio看板提供统计数据源；
    - 素材标签分类、历史绘图记录检索、任务数据Excel导出；
3. 数据归属约束：所有用户、运营相关数据仅存储在UcubMQDraw，不传递至底层imggen。

### 4.3 MQ消息调度层（阿里云RocketMQ 按量付费标准版）
1. 队列划分（流量隔离，区分两种算力服务）
    - `ucub_imggen_api_task`：ComfyUI API并发版消费队列
    - `ucub_imggen_gpt_task`：ComfyUI GPT串行版消费队列
    - `ucub_imggen_callback`：统一结果回调队列（两种算力服务共用）
2. 部署配置：仅开启VPC内网接入，关闭公网访问；消息保留时长设置30分钟，降低存储计费；
3. 计费成本：月度前2000万次API调用免费，30人内部流量完全覆盖；仅收取Topic日占用费，月度总成本60~70元。

### 4.4 底层算力推理层：imggen（双版本ComfyUI服务化微服务）
#### 公共底层能力（两个版本共用）
1. 内置ComfyUI工作流解析引擎：支持导入ComfyUI导出的JSON工作流，自动转换为节点可执行代码；
2. 仅处理纯绘图参数，**不识别、不存储、不传递任何用户/运营业务数据**；
3. 绘图完成后自动将生成图片上传阿里云OSS，仅返回图片地址；
4. 内置异常捕获、推理超时捕获，失败时携带错误信息发送回调消息；
5. 仅内网消费RocketMQ队列，无公网端口暴露。

#### 版本1：imggen-ComfyUI-API 并发版
1. 资源特性：无单实例显卡串行限制，支持多队列、多工作流并发同时执行绘图任务；
2. 适用场景：大批量商品图、简单文生图、轻量ComfyUI工作流，追求高吞吐；
3. 消费队列：`ucub_imggen_api_task`；
4. 优势：横向扩展多实例可无限提升并发，适合运营批量作图。

#### 版本2：imggen-ComfyUI-GPT 串行版
1. 资源特性：依赖GPU显卡推理资源，单实例同一时间**仅能串行执行1个任务**，新任务排队等待上一个完成；
2. 适用场景：带AI图文理解、长提示词优化、复杂多节点ComfyUI工作流；
3. 消费队列：`ucub_imggen_gpt_task`；
4. 约束：多任务同时下发时，imggen内部自动排队，逐一生成。

### 4.5 统一存储层
1. MySQL 8.0：结构化业务数据（用户、任务、ID映射、素材、统计、工作流模板）；
2. 阿里云OSS：非结构化文件（运营上传参考图、imggen生成成品图、ComfyUI工作流文件）；
3. RocketMQ临时存储：仅缓存排队任务，30分钟自动清理。

## 五、完整业务执行流程
### 流程1：运营提交绘图任务（API并发版举例，GPT版本逻辑一致）
1. 运营在Gradio页面填写提示词、上传参考图、选择`comfy_api`算力类型、绑定ComfyUI工作流模板，提交任务；
2. FastAPI接收完整请求参数，生成业务唯一taskId，将userId、操作参数、素材信息存入任务主表，任务状态置为「排队中」；
3. 平台生成独立innerTaskId，写入`task_inner_mapping`映射表；
4. 过滤剔除userId、账号、运营标签等业务字段，组装纯净算力消息体；
5. 根据computeType将消息投递至对应队列`ucub_imggen_api_task`；
6. Gradio前端通过WebSocket/轮询查询任务状态，实时展示排队进度。

### 流程2：imggen-ComfyUI-API 并发版消费执行绘图
1. imggen持续监听`ucub_imggen_api_task`，拉取消息，仅解析绘图参数与innerTaskId；
2. 内置引擎解析消息内ComfyUI工作流JSON，转换为可执行节点代码；
3. 多并发调度GPU资源并行执行多张图像推理；
4. 图像生成完成后批量上传OSS，获取文件访问地址；
5. 组装仅包含innerTaskId、状态、图片地址、耗时的回调消息，发送至`ucub_imggen_callback`；
6. 若推理报错/超时，status置为fail，填充errorMsg，同样发送回调消息。

### 流程3：imggen-ComfyUI-GPT 串行版执行差异
1. 消费`ucub_imggen_gpt_task`队列消息；
2. 单实例内部任务串行排队，同一时间仅运行1条绘图任务；
3. 其余工作流解析、绘图、OSS上传、回调逻辑与API版完全一致。

### 流程4：UcubMQDraw 接收回调更新任务
1. 平台消费统一回调队列`ucub_imggen_callback`；
2. 使用innerTaskId查询映射表，匹配业务taskId；
3. 更新MySQL任务主表状态、填充图片OSS地址、记录生成耗时/错误信息；
4. WebSocket推送状态至运营Gradio页面，自动展示生成图片；
5. 失败任务在页面提供「一键重跑」按钮，重跑流程复用完整提交逻辑。

## 七、安全设计方案
1. **算力网络隔离**：imggen显卡服务器全部关闭公网端口，仅内网连通阿里云MQ；
2. **MQ访问限制**：RocketMQ实例关闭公网接入，仅VPC内网地址可生产/消费消息；
3. **业务数据隔离**：用户ID、运营账号等敏感业务数据仅存在UcubMQDraw，不向下传递至imggen；
4. **消息轻量化**：MQ不传输图片二进制，仅传递OSS文件路径，降低数据泄露风险；
5. **权限管控**：运营账号数据隔离，仅可见本人提交任务，管理员独立权限；
6. **任务容错**：MQ消息支持重试，算力崩溃任务可一键重跑，避免绘图任务丢失。

## 八、成本测算（仅阿里云RocketMQ）
1. 计费模式：按量付费标准版，无公网带宽费用；
2. API调用费：30人内部每日任务量极低，月度调用量远低于2000万免费额度，调用费用0元；
3. Topic占用费：3个业务Topic，单Topic 2元/天；
4. 存储成本：消息仅留存30分钟，每月存储费5~15元；
5. 月度MQ总预估成本：180 ~ 200元。
